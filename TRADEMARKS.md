# Trademark & Naming Policy

The Agent Runtime source code is licensed under [Apache 2.0](LICENSE). **That
license covers the code — it does not grant rights to the project's names, logos,
or other brand assets.** Apache 2.0 §6 expressly reserves trademarks; this file
explains what that means in practice.

## What this covers

The project names and any associated logos or wordmarks (collectively, the
"Marks") — including "Agent Runtime" and "ARIA" as used by this project.

## What you may do (no permission needed)

- **Nominative use.** Truthfully state that your work uses, integrates with, is
  based on, or is compatible with Agent Runtime (e.g., "an MCP server for Agent
  Runtime", "a fork of Agent Runtime").
- **Redistribute** the software under Apache 2.0, keeping `LICENSE` and `NOTICE`
  intact.
- **Discuss, teach, review, and reference** the project freely.

## What needs permission

- Using the Marks as the name of **your** product, service, fork, or company, or
  in any way that implies the project **endorses, sponsors, or is affiliated
  with** your offering.
- Modifying the Marks, or using confusingly similar names or logos.
- Using the Marks on merchandise.

A fork distributed in a way that could confuse users about its origin must be
**renamed**: keep the nominative "based on Agent Runtime" attribution, but drop
the Mark as the product name.

## No active trademark registration

The project does not currently hold a registered trademark. This policy documents
intent and the Apache 2.0 §6 reservation; it is not a claim of registration, and
it may be updated as the project matures.

## Requests & questions

Email **ardenagentic+trademark@gmail.com**. Good-faith community and nominative
use is the norm — this policy exists to prevent confusion, not to restrict
legitimate use.
