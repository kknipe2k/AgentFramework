ARIA

Agentic Rail-based Intent Architecture

An orchestration system for autonomous AI-assisted development with safety rails

What is ARIA?

ARIA is an orchestration layer that sits between you and an AI coding assistant. It adds structure, verification, and human oversight to autonomous development.

Rails, Not Rules

Safety guardrails that guide without blocking. The AI stays productive while respecting boundaries.

Right-Sized Process

A 5-line bug fix shouldn't need the same ceremony as a new feature. ARIA adapts its process to task complexity.

Human-in-the-Loop

Strategic checkpoints before risky actions. You stay in control without micromanaging every step.

Full Traceability

Every decision logged with rationale. Know why the AI did what it did, not just what changed.

The Problem ARIA Solves

Without ARIA:

  • AI makes changes without verification → bugs ship
  • No stopping point for risky actions → unintended consequences
  • No record of decisions → "why did it do that?"
  • Same process for simple and complex tasks → inefficient

With ARIA:

  • Mandatory verification after every code change
  • HITL checkpoints before deletions, config changes, risky actions
  • Decision trace: action → context → rationale → alternatives → confidence
  • LITE/STANDARD/FULL modes scale process to task size

When HITL Triggers

File Deletions

Any file deletion requires explicit approval.

Config Changes

package.json, tsconfig, CI/CD files trigger checkpoints.

"Don't Touch" Areas

Files marked in project-context.md require approval.

New Dependencies

Installing new packages triggers review.

Decision Traces

Every significant decision is logged with full context. Here's a real trace:

🔧
Use existing retry pattern from utils/retry.ts
Context: Read src/utils/retry.ts, found 3 similar implementations
Rationale: Consistency with existing codebase patterns
Alternatives: Custom retry implementation, No retry
Confidence: 0.85
🛡️
Add input validation before API call
Context: Security review, OWASP guidelines
Rationale: Prevent injection attacks and malformed input
Alternatives: Trust caller, Validate server-side only
Confidence: 0.92
⏭️
Skip edge case tests for now
Context: Time constraint, happy path working
Rationale: Core functionality verified, edge cases can follow
Alternatives: Full test coverage, Delay feature
Confidence: 0.60

Why Traces Matter

  • Debugging: When something goes wrong, understand the reasoning
  • Learning: Improve the AI's decisions over time
  • Compliance: Audit trail for regulated industries
  • Handoff: New team members understand why code looks the way it does
