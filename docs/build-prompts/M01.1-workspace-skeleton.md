## Problem statement

The repo has a complete spec, schemas, examples, and OSS scaffolding. It has zero Rust code. The CI workflow's Rust jobs are gated on `Cargo.toml` existing and currently skip.

M01.1 unblocks every subsequent sub-milestone by establishing the **Cargo workspace skeleton** with:
- Empty member crates (`runtime-core`, `runtime-main`, `runtime-drone`, `runtime-sandbox`, `xtask`)
- Tauri stub (`src-tauri/`) with empty webview
- Workspace-wide lint configuration per §12 (deny warnings, forbid unsafe except sandbox, clippy pedantic + nursery)
- `cargo deny` policy file
- CI green on Linux/macOS/Windows × stable/MSRV with the new Rust jobs active

**No real implementation in any crate.** Each crate has only `pub fn placeholder() -> &'static str { "ok" }` and a single trivial test confirming the test runner executes. M01.2 adds type generation to `runtime-core`; M01.3 adds the drone implementation. This sub-milestone is the foundation-foundation.

The success criterion is "CI is green on a real Cargo workspace and the next sub-milestone can begin."

## Code expectations

### Workspace `Cargo.toml` shape

```toml
[workspace]
resolver = "2"
members = [
    "crates/runtime-core",
    "crates/runtime-main",
    "crates/runtime-drone",
    "crates/runtime-sandbox",
    "crates/xtask",
    "src-tauri",
]

[workspace.package]
version = "0.1.0"
edition = "2021"
license = "Apache-2.0"
repository = "https://github.com/kknipe2k/AgentFramework"
rust-version = "1.80"

[workspace.dependencies]
# Empty for M01.1 — populated by later sub-milestones
# Keep this section to establish the pattern: shared deps go here

[workspace.lints.rust]
unsafe_code = "forbid"
warnings = "deny"

[workspace.lints.clippy]
pedantic = { level = "warn", priority = -1 }
nursery  = { level = "warn", priority = -1 }
```

`runtime-sandbox`'s `Cargo.toml` overrides `unsafe_code` to `warn` (later milestones add unsafe blocks there with `// SAFETY:` comments).

### Per-crate `Cargo.toml` shape

```toml
[package]
name = "runtime-core"
version.workspace = true
edition.workspace = true
license.workspace = true
repository.workspace = true
rust-version.workspace = true

[lints]
workspace = true
```

### `lib.rs` template (per crate)

```rust
//! Runtime <crate-name> placeholder.
//!
//! Real implementation lands in subsequent M01 sub-milestones.

/// Returns the string `"ok"`. Placeholder for M01.1; real exports come later.
///
/// # Examples
///
/// ```
/// assert_eq!(runtime_core::placeholder(), "ok");
/// ```
#[must_use]
pub fn placeholder() -> &'static str {
    "ok"
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn placeholder_returns_ok() {
        assert_eq!(placeholder(), "ok");
    }
}
```

### What NOT to write

- **Real type definitions.** Don't pre-create `AgentEvent` or `DroneEvent` in M01.1 — they belong to M01.2 where `runtime-core` becomes the curated event-taxonomy crate.
- **Dependencies beyond what's needed.** No `serde`, `tokio`, `clap`, `thiserror` in M01.1. The crates compile with stdlib only.
- **Tauri allowlist entries.** `tauri.conf.json` ships with an empty allowlist; M02 adds the first command.
- **Module hierarchy.** `lib.rs` is a single file; no `mod` declarations until later sub-milestones add real code.

## Self-correction guidance

Likely failure modes for M01.1:

| Failure | Likely cause | Fix |
|---|---|---|
| `cargo build` fails on Tauri stub | Missing platform deps (webkit2gtk on Linux, etc.) | Follow Tauri prerequisites docs; CI runners have most installed already |
| `cargo deny check` fails on first run | Missing `deny.toml` or default policy too strict | Start with permissive policy + tighten incrementally; the rules in scope are the target, not day-1 |
| `cargo doc` warns on missing docs | Placeholder fns missing `///` comments | Add doc comment with example per the template above |
| Clippy pedantic complains about placeholder fn | Pedantic is opinionated; allow specific lints with rationale | Use `#[allow(clippy::missing_const_for_fn)]` etc. with linked-issue comment |
| CI Windows-only failure | Path separator or `tauri.conf.json` line endings | Inspect CI log; usually resolved by `.gitattributes` enforcing LF |

Per `CLAUDE.md` §7, escalate after 3 self-correction iterations.

## Sub-milestone-specific gotchas

1. **Tauri prereqs vary by OS.** Linux needs webkit2gtk; macOS needs Xcode CLI tools; Windows needs WebView2 (often pre-installed). CI runners have most. If a local build fails on prereqs, document the fix in `crates/runtime-drone/README.md` (created in M01.4).
2. **`Cargo.lock` should be committed for binary projects** (this is one). Don't `.gitignore` it.
3. **Workspace lints require Rust 1.74+.** The `[workspace.lints]` and `[lints]` tables aren't available before that. M01.1's `rust-toolchain.toml` should pin ≥1.80, well above this threshold.
4. **`cargo deny` needs the policy file in repo root.** Default location is `deny.toml`; alternative is `.cargo/deny.toml`. Use repo root for visibility.
5. **MSRV in CI.** §12 specifies stable + MSRV (1.80). The MSRV cell tests that the project still builds on the oldest supported Rust; for M01.1 with no deps, this is trivially true. Keep the cell active so it catches regressions later.
