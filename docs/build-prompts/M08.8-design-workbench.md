# M08.8 — Design Workbench + Gap-Resume

> **The milestone where execution becomes app-observable and the runtime becomes human-operable.** M08.7a made the single-agent engine *execute* (rungs 1–5, observed in the assembled engine + per-rung IRL); the app still can't *show* it. M08.8 builds the front-to-back workbench — new-load → run → **watch** → gap → **Grant/Decline** → resume → complete — to `DESIGN.md`. Third beat of the arc: **paints** (pre-M08.7) → **executes** (M08.7) → **observable + operable** (M08.8).

---

## Background — the grounded state at entry

**What runs but isn't visible.** The execution-status ledger (`docs/execution-status.md`) records rungs 1–5 as `executes — observed` **at the engine granularity** (the assembled `run_test_session_with` evals E-01..E-05 + per-rung IRL). The assembled *app* was not the observation surface for any of them:

- **Execution is log-only.** Tool calls / agent text / completions surface through `log_event_debug` at `RUST_LOG=debug`, never on the live graph in the running app. That is **TD-034** (agent output not visible in the app).
- **Budget enforces but is invisible.** Rung 5 halts a run at the cap, but `BudgetHeaderBar` reads the *live* `useGraphStore` while a Tester run replays into the *scoped* `useTestGraphStore` (ADR-0019 isolation) — so the header is structurally blind to a Tester run's spend; and `log_event_debug` drops every `Budget*` event. The rung-5 in-app observe was explicitly deferred here.
- **The gap suspends but dead-ends.** Rung 4 suspends cleanly and records a recoverable snapshot, but there is no resolution surface — the human cannot Grant/Install/Decline and resume. ADR-0029 (Accepted) pulled resolve-and-resume into v0.1; its **engine resume** + **resolution UI** land here.
- **The Builder carries 32 IRL findings** (`docs/M08.6-irl-findings.md`), 5🔴 of which routed to this milestone (#12 clear-canvas, #17 save-from-scratch, #19 tier-UI, #22 budget-cap-persistence, #32 save-companions) plus the 🟡/🟢 UX clusters.

**The unifying insight.** `DESIGN.md`'s nine interaction principles **each trace to a Builder finding** (principle 1 → the no-feedback cluster #1/#14/#21 incl. the dead "Save Cap"; principle 5 → recoverability #11/#12/#13; principle 8 → the "Promote to Promoted" mislabel #20; the component-modal rule → #23/#24). **Building the app *to `DESIGN.md`* IS how the findings get resolved** — the visual layer and the findings-fix are one body of work, not two. So the findings are not a back-half dump; each rides the stage that owns its surface.

**The standard this milestone is held to (rule 11 + ADR-0021).** A UI surface "works" only when the **real Tauri app** has been driven through `tauri-driver` and the behavior observed — never Playwright-mock-green, never code-read. Every stage closes on the real-app IRL. The execution-status ledger gains an **"observed in app"** bar (distinct from M08.7's engine-observe) that M08.8 stages flip.

**Inputs (read at each stage's entry):**

- `DESIGN.md` — the committed "Light Instrument" visual layer (#117): the nine interaction principles, the node-kind two-axis model, the component specs (the live-graph execution view is the centerpiece), modals/panels/badges/toasts behavior rules, AA-contrast tokens.
- `docs/adr/0029-gap-resolve-and-resume.md` (Accepted) — the full gap loop: engine resume (reload snapshot → deliver resolved `tool_result` → continue; **idempotent rebuild §1b — no re-execution of completed calls — is the load-bearing assertion**) + the **Grant / Install / Decline** governed-elevation UI.
- `docs/adr/0021-*` — the `tauri-driver` real-app E2E gate (the merge-blocking close bar).
- `docs/tech-debt.md` — **TD-034** (agent output in app), **TD-036** (tier never wired into the run loop — Tester/smoke always Novice). **TD-037** (canvas authors no skill bodies) **stays M09 Generators** — out of M08.8 scope.
- `docs/M08.6-irl-findings.md` — the 32 findings + clusters; the 5🔴 routed here.
- `docs/execution-status.md` — the paints→executes ledger; stages add the "observed in app" status.

## Scope

**In scope (v0.1 §0d holds):** the app surfaces that make the *already-shipped* engine observable + operable — the live-graph execution view, the budget surface, the gap-resolution UI + the engine resume, tier-in-the-run-loop, and the Builder-findings UX build-out — all to `DESIGN.md`, all real-app IRL'd.

**Out of scope:** new engine primitives (sub-agents / plan / hooks / MCP-fix / composition = **M08.9**); canvas skill-body authoring (**TD-037 = M09 Generators**); the mentor (**M09**, deprioritized). No new execution capability is built here — M08.8 *exposes and operates* what M08.7 built; the only new *engine* work is ADR-0029's gap **resume** (the operability half of an already-Accepted decision).

**Locks:** built to `DESIGN.md` (a surface diverging from it is a Stage-D design-review finding); real-app `tauri-driver` IRL is the close bar (ADR-0021); no schema change unless a stage proves one (ADR per §11); findings distributed by surface affinity, not dumped.

## Stage A — Live-graph execution view (TD-034) — the observability foundation

### A.1 Problem statement

A real run's agent output is **invisible in the app** (TD-034, maintainer-confirmed 2026-05-31). Agent events already reach the live graph — `src/lib/ipc.ts:641` `listen<AgentEvent>('agent_event', …)` → `graphStore.applyEvent` (`App.tsx:143`); the backend emits each event on the `agent_event` Tauri channel (`src-tauri/src/commands.rs:297/584` `app.emit("agent_event", &event)`) — and the events **paint nodes**. But the agent's *output* is unwatchable:

- `stream_text` is a **`graphStore` no-op** ("feeds the inspector, not the [graph]" — `graphStore.ts:1574,1734`) that renders **nowhere**. The agent's reply text never appears in the app.
- Tool nodes store `toolName` (`graphStore.ts:902,949`) but **no input/result payload**, and nodes aren't clickable — so a `ToolInvoked(Read)` / `ToolResult` node exposes nothing (you can't see what was read or returned).

So a run paints shapes a human can't read; the only observe channel is `RUST_LOG`. M08.8.A makes execution observable **in-app**, to DESIGN.md's Inspector/Output rail — without a debug log. It is the foundation B–E are watched through.

### A.2 Files to change

- **A new Output/Inspector rail** — DESIGN.md's 360px right-rail "Inspector / Output" (`DESIGN.md` Layout & spacing): renders the streamed agent text live + the selected node's payload, **IBM Plex Mono** for machine output (DESIGN.md Typography "instrument register").
- `src/lib/graphStore.ts` + `src/lib/builderStore.ts` (the scoped test-graph store) — surface `stream_text` (currently no-op) into an **output buffer** the rail reads; **retain tool `{input, output}` payloads** on nodes for inspection.
- `src/components/builder/TesterGraphPane.tsx` (the primary candidate-run watch surface) + `src/components/GraphCanvas.tsx` (the live session) — node click → selection feeding the Inspector; node render to the DESIGN.md two-axis kind/status model.
- `tests/e2e-tauri/execution_view.e2e.ts` (new — the real-app close) + a vitest/Playwright unit (the rail renders given a mocked event stream).
- **No schema change** — the `AgentEvent` union already carries `text` / tool `input` / `output_preview`.

### A.3 Detailed changes

1. `stream_text` → append `event.text` to a per-session **output buffer** in the store; the Output rail renders it live as the agent's reply (DESIGN principle 1 — visible output; mono register).
2. Tool nodes **retain `{input, output}`**; clicking a node selects it; the Inspector renders the selected node's payload (the `Read` input path + the file contents that fed back) — closing the "nodes not clickable / no payload" half of TD-034.
3. Nodes render per DESIGN.md **two-axis** model (kind: accent bar + glyph chip `A/T/S/H/‖`; status: ring/dot idle→active→complete/error/gap) to the extent a single-agent built-in run surfaces them (Agent + Tool; active→complete).
4. Wired for **both** the Tester scoped graph (`useTestGraphStore`) and the live `useGraphStore` — the same rail/inspector component over either store.

### A.4 Acceptance (BDD — the close contract)

```gherkin
Feature: A run is observable in the running app
  Scenario: watch a built-in-tool run
    Given a framework whose agent reads a file and quotes it
    When I run it in the app (Tester)
    Then the agent's streamed reply text is visible in the Output rail (not only RUST_LOG)
    And clicking the Read tool node shows its input path + the file contents it returned
    And the Tool node renders teal (kind) and green (complete status) per DESIGN.md
```

Asserted on the **real Tauri app via `tauri-driver`** (`execution_view.e2e.ts`) + the maintainer IRL watch — NOT Playwright-mock-green, NOT code-read (rule 11 / ADR-0021).

### A.5 construction_reachability_check

Dead wires today (both `inputs_reachable="false"` on `main`): `stream_text` is dropped at `graphStore.ts:1734` → the Output rail has nothing to render; tool nodes carry no payload → a click exposes nothing. Stage A inverts both to `"true"` (text buffered + rendered; payloads retained + inspectable). The build greps `case 'stream_text'` pre-A (no-op) vs post-A (appends to the output buffer the rail reads) and confirms the inversion.

### A.6 Close gate

- **Strict v1.8 two-commit TDD** — red: the e2e + unit asserting output-visible fail for the right reason; impl without touching the test files (`git diff <red>..<impl>` over test paths EMPTY).
- **Machine gates** — `prettier --check` / `eslint` / `tsc --noEmit` / `vitest` (≥80% `src/`) + **`npm run test:e2e:tauri`** (the real-app close, ADR-0021; the merge-blocking `e2e-tauri-driver` job).
- **Mutation gate** — advisory (renderer observability is not execution-wiring / a §6 critical module per `cluster-pattern.md` §5).
- **The close flips execution-status's new "observed in app" status** for the execution-view surface — on the maintainer IRL watch, not tests-green (rule 11).
- **Stage-D design-review** — the Output rail + node render match DESIGN.md (the Inspector/Output rail, the node-kind two-axis model, the mono register, AA contrast). A divergence is a design-review finding.
