# M08.8 — Design Workbench + Gap-Resume

> **The milestone where execution becomes app-observable and the runtime becomes human-operable.** M08.7a made the single-agent engine *execute* (rungs 1–5, observed in the assembled engine + per-rung IRL); the app still can't *show* it. M08.8 builds the front-to-back workbench — new-load → run → **watch** → gap → **Grant/Decline** → resume → complete — to `DESIGN.md`. Third beat of the arc: **paints** (pre-M08.7) → **executes** (M08.7) → **observable + operable** (M08.8).

---

## Background — the grounded state at entry

**What runs but isn't visible.** The execution-status ledger (`docs/execution-status.md`) records rungs 1–5 as `executes — observed` **at the engine granularity** (the assembled `run_test_session_with` evals E-01..E-05 + per-rung IRL). The assembled *app* was not the observation surface for any of them:

- **Execution is log-only.** Tool calls / agent text / completions surface through `log_event_debug` at `RUST_LOG=debug`, never on the live graph in the running app. That is **TD-034** (agent output not visible in the app).
- **Budget enforces but is invisible.** Rung 5 halts a run at the cap, but `BudgetHeaderBar` reads the *live* `useGraphStore` while a Tester run replays into the *scoped* `useTestGraphStore` (ADR-0019 isolation) — so the header is structurally blind to a Tester run's spend; and `log_event_debug` drops every `Budget*` event. The rung-5 in-app observe was explicitly deferred here.
- **The gap suspends but dead-ends.** Rung 4 suspends cleanly and records a recoverable snapshot, but there is no resolution surface — the human cannot Grant/Install/Decline and resume. ADR-0029 (Accepted) pulled resolve-and-resume into v0.1; its **engine resume** + **resolution UI** land here.
- **The Builder carries 32 IRL findings** (`docs/M08.6-irl-findings.md`), 5🔴 of which routed to this milestone (#12 clear-canvas, #17 save-from-scratch, #19 tier-UI, #22 budget-cap-persistence, #32 save-companions) plus the 🟡/🟢 UX clusters.

**The unifying insight.** `DESIGN.md`'s nine interaction principles **each trace to a Builder finding** (principle 1 → the no-feedback cluster #1/#14/#21 incl. the dead "Save Cap"; principle 5 → recoverability #11/#12/#13; principle 8 → the "Promote to Promoted" mislabel #20; the component-modal rule → #23/#24). **Building the app *to `DESIGN.md`* IS how the findings get resolved** — the visual layer and the findings-fix are one body of work, not two. So the findings are not a back-half dump; each rides the stage that owns its surface.

**The standard this milestone is held to (rule 11 + ADR-0021).** A UI surface "works" only when the **real Tauri app** has been driven through `tauri-driver` and the behavior observed — never Playwright-mock-green, never code-read. Every stage closes on the real-app IRL. The execution-status ledger gains an **"observed in app"** bar (distinct from M08.7's engine-observe) that M08.8 stages flip.

**Inputs (read at each stage's entry):**

- `DESIGN.md` — the committed "Light Instrument" visual layer (#117): the nine interaction principles, the node-kind two-axis model, the component specs (the live-graph execution view is the centerpiece), modals/panels/badges/toasts behavior rules, AA-contrast tokens.
- `docs/adr/0029-gap-resolve-and-resume.md` (Accepted) — the full gap loop: engine resume (reload snapshot → deliver resolved `tool_result` → continue; **idempotent rebuild §1b — no re-execution of completed calls — is the load-bearing assertion**) + the **Grant / Install / Decline** governed-elevation UI.
- `docs/adr/0021-*` — the `tauri-driver` real-app E2E gate (the merge-blocking close bar).
- `docs/tech-debt.md` — **TD-034** (agent output in app), **TD-036** (tier never wired into the run loop — Tester/smoke always Novice). **TD-037** (canvas authors no skill bodies) **stays M09 Generators** — out of M08.8 scope.
- `docs/M08.6-irl-findings.md` — the 32 findings + clusters; the 5🔴 routed here.
- `docs/execution-status.md` — the paints→executes ledger; stages add the "observed in app" status.

## Scope

**In scope (v0.1 §0d holds):** the app surfaces that make the *already-shipped* engine observable + operable — the live-graph execution view, the budget surface, the gap-resolution UI + the engine resume, tier-in-the-run-loop, and the Builder-findings UX build-out — all to `DESIGN.md`, all real-app IRL'd.

**Out of scope:** new engine primitives (sub-agents / plan / hooks / MCP-fix / composition = **M08.9**); canvas skill-body authoring (**TD-037 = M09 Generators**); the mentor (**M09**, deprioritized). No new execution capability is built here — M08.8 *exposes and operates* what M08.7 built; the only new *engine* work is ADR-0029's gap **resume** (the operability half of an already-Accepted decision).

**Locks:** built to `DESIGN.md` (a surface diverging from it is a Stage-D design-review finding); real-app `tauri-driver` IRL is the close bar (ADR-0021); no schema change unless a stage proves one (ADR per §11); findings distributed by surface affinity, not dumped.

## Process (per stage)

1. **Authored at entry** against the live renderer + the `DESIGN.md` component spec (§8 phase-doc pre-flight — cross-machine state checked).
2. **Cluster-gate close discipline** (`docs/cluster-pattern.md`): acceptance-first (BDD) → strict-TDD two-commit → machine gates + mutation gate (on any run-loop/enforcer touch, e.g. Stage C resume, Stage D tier) → **RUN the real app + the maintainer IRL** → triage-in-place (zero-propagation).
3. **Built to `DESIGN.md` + a Stage-D design-review** — the first design-review run of the project (the scaffolding in ORCHESTRATOR open-threads); a surface diverging from `DESIGN.md` is a design-review finding, not shipped.
4. **The close flips the execution-status "observed in app" status** for the surface — on the real-app IRL observation, never tests-green (rule 11).
5. **E2E is `tauri-driver` against the built binary** (ADR-0021; the merge-blocking `e2e-tauri-driver` job). Playwright-renderer is structurally blind to the Tauri shell — it does not close a stage.
