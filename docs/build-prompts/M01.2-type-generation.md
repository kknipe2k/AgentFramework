## Problem statement

M01.1 landed empty crates. M01.2 makes `runtime-core` the curated **type vocabulary** for the project:

1. **Generated types** from `schemas/*.v1.json` via `typify` — `Framework`, `Skill`, `Tool`, `Agent`, `Capabilities`, `Provenance`, `HookRef`, `Hook`, `JSONLogicExpression`, etc. Source of truth lives in `schemas/`; CI fails if committed types differ from regenerated.
2. **Hand-curated event taxonomy** — `AgentEvent`, `DroneEvent`, `DroneCommand`, plus supporting enums (`ActivityState`, `StopReason`, `ProcessType`, `AlertLevel`, `RevertReason`). These are NOT generated; they're the contract every later milestone evolves. Defined once at M01.2; minor additions are minor version bumps; removals require ADRs.
3. **`xtask regenerate-types` subcommand** — the build tool that runs typify against schemas and writes to `crates/runtime-core/src/generated/`. Includes a `--check` mode for CI drift detection.

After M01.2, M01.3's drone can serialize `DroneEvent` and deserialize `DroneCommand` against the canonical types. M02's main process can emit `AgentEvent` against the canonical union.

### In scope

- `tool.rs` from `schemas/tool.v1.json`
- `agent.rs` from `schemas/agent.v1.json`

**Hand-curated types in `crates/runtime-core/src/`:**
- `event.rs` — `AgentEvent` enum with ALL variants from spec §2 + §2a + §2b + §3a + §3b + §4a + §4b + §6a + §8.security. Use `#[serde(tag = "type", rename_all = "snake_case")]`. Variants that aren't yet emitted are still valid Rust at M01.2; they're emitted by later milestones.
- `drone.rs` — `DroneEvent` and `DroneCommand` enums per spec §1d, with `ActivityState`, `StopReason`, `ProcessType`, `ProcessConfig`, `RevertReason` supporting types.
- `error.rs` — `RuntimeError` via `thiserror` for type-loading and serde failures.
- `lib.rs` — re-exports from `generated/`, `event`, `drone`, `error` for downstream crates.

**xtask implementation in `crates/xtask/src/`:**
- `main.rs` with clap subcommands: `regenerate-types` and `regenerate-types --check`
- `regenerate-types`: reads `schemas/*.v1.json`, runs typify, writes to `crates/runtime-core/src/generated/`
- `regenerate-types --check`: regenerates to a temp dir, diffs against committed; exits non-zero if drift detected
- Documented error paths (schema parse error, typify error, write error)

**CI integration:**
- Update `.github/workflows/ci.yml` to add a `cargo xtask regenerate-types --check` step (gates on `Cargo.toml` existing; already handled by M01.1's setup). Drift = CI fails.

**TS frontend types** are **NOT** part of M01.2 — they land at M03 when the renderer exists. M01.2 is Rust-side only.

### Out of scope

- **Runtime drone implementation** — M01.3
- **Provider abstractions, AgentSdk** — M02
- **TS type generation** (`json-schema-to-typescript`) — M03 alongside the renderer
- **Manually editing generated files** — never. If typify output is wrong, fix it via post-processing in `xtask`, not by editing `generated/*.rs`. CI's drift check enforces this.

## Code expectations

### Generated file template (auto-emitted by `xtask`)

```rust
// AUTO-GENERATED FILE — DO NOT EDIT
//
// Regenerate with: `cargo xtask regenerate-types`
// Source schema: schemas/<name>.v1.json
// Generated by: typify <version>
//
// Drift between this file and the schema is detected by CI
// (`cargo xtask regenerate-types --check`).

use serde::{Deserialize, Serialize};

// <typify output here>
```

### Hand-curated `event.rs` shape (excerpt)

```rust
//! AgentEvent — the canonical event union emitted by the runtime.
//!
//! Variants span the spec sections they belong to; cross-references in
//! variant doc comments link back to the originating spec section.

use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Serialize, Deserialize, PartialEq)]
#[serde(tag = "type", rename_all = "snake_case")]
pub enum AgentEvent {
    /// Session lifecycle. Spec §2.
    SessionStart {
        session_id: String,
        framework: String,
        model: String,
    },
    /// Agent spawned. Spec §2.
    AgentSpawned {
        agent_id: String,
        agent_name: String,
        parent_id: Option<String>,
    },
    // ... (50+ variants total per spec)
}
```

### What NOT to write

- **Hand-edited generated files.** Treat `generated/` as derived output. If the type is wrong, fix it in `xtask` (post-processing) or in the schema (source of truth).
- **Custom serde impls.** Use derives + `#[serde(...)]` attributes only. Hand-rolled `Serialize`/`Deserialize` is forbidden in M01.2.
- **Variants you can't justify against the spec.** Every `AgentEvent` variant must trace to a spec section. If you'd add one without a section reference, surface it instead.

## Self-correction guidance

| Failure | Likely cause | Fix |
|---|---|---|
| typify panics on `oneOf` with discriminator | typify limitation on certain JSON Schema constructs | Post-process in `xtask`; or simplify the schema if possible (with ADR if it's a structural change) |
| Generated type doesn't deserialize an example | Schema constraint missed by typify (e.g., `additionalProperties: false`) | Add `#[serde(deny_unknown_fields)]` via post-processing; or accept the looseness if not security-critical |
| `cargo xtask regenerate-types --check` fails right after manual edit | You hand-edited a generated file (forbidden) | Revert; encode the change in `xtask` post-processing or in the schema |
| `AgentEvent` round-trip fails on one variant | Tag attribute wrong | Confirm `#[serde(tag = "type", rename_all = "snake_case")]` on the enum; variant name should be `PascalCase` and serialize as `snake_case` |
| `examples/aria/skills/planning.md` deserializes but loses fields | Frontmatter has fields the schema doesn't declare | Add the field to the schema (and regenerate) OR document it as intentional extension |

Per CLAUDE.md §7, escalate after 3 self-correction iterations.

## Sub-milestone-specific gotchas

1. **typify version pin.** Pin a specific typify version in `xtask/Cargo.toml`. Different versions can produce different output → drift check would falsely fail.
2. **Schema $ref resolution.** typify needs to resolve `$ref` to local files (`common.v1.json#/$defs/Capabilities`). The xtask should pass schemas to typify in the right order or use a base URI.
3. **Typify output is opinionated.** Generated names use Rust conventions (CamelCase for types, snake_case for fields). The schema's JSON-style names get translated. Document the translation table in `runtime-core/README.md` if it's non-obvious.
4. **The `AgentEvent` enum is large** (50+ variants by spec). Don't try to test every variant individually; the round-trip property test (#11) covers them combinatorially. Add explicit unit tests only for variants whose tag attribute is non-standard.
5. **Doc comments on generated code.** typify emits doc comments from the schema's `description` fields; these become rustdoc. Verify `cargo doc` runs cleanly without `missing_docs` warnings.
