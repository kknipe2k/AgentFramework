## What M08.7a shipped (the one-paragraph product)

M08.7 is the milestone that answers the load-bearing M08.6-IRL finding — *the
runtime paints most of what it claims; only provider-streaming + MCP execute*.
The eval-first ladder (`docs/cluster-pattern.md` §1) closes five execution
primitives, each isolated against a minimal hand-authored framework, each
flipping its `docs/execution-status.md` row from `paints` to `executes —
observed` only when the **assembled** path ran and the **side effect** was
seen (CLAUDE.md §4 rule 11):

- **Rung 1 / Stage A** — built-in tool execution. `Read`/`Write` (plus
  `Glob`/`Grep`/`WebFetch`) run in-process and feed their result back into the
  agent's next turn through `TurnFeedback::dispatched` (the wire that
  distinguishes "executed and fed back" from "painted and dropped").
- **Rung 2 / Stage B** — capability enforcement gates a *live* tool. An
  out-of-scope `Write` emits `CapabilityViolation` and the side effect never
  happens (no file on disk); an in-scope `Write` writes it.
- **Rung 3 / Stage C** — `LoadSkill` injects the skill body into context (ADR-0027:
  the body rides back as the `LoadSkill` `tool_result`, persisting in
  `config.messages` across turns) and the agent's behavior changes.
- **Rung 4 / Stage D** — `request_capability` gap detection wired into the run
  loop → clean **suspend-and-record** (loop-break, snapshot, recoverable per
  §1b). The **resume half** was pulled into v0.1 as a scheduled M08.9 gap-resume
  rung (ADR-0029), not built here.
- **Rung 5 / Stage E** — budget enforcement at the run loop: per-turn
  `record_spend` → warn / downshift / hard-stop, idempotent across turns.

The milestone produced **ADR-0028** (the built-in tool execution contract — the
archetype every later rung mirrors; filed at closeout per V's 🟡 #1) and
**ADR-0027** (skill-into-context injection). M08.7.V (the five-pass verifier,
including the v1.9 5th `assembled_execution` pass) returned **Sound, 0🔴** with
one 🟡 (routed → ADR-0028) and two 🟢 (routed → TD-039 / TD-040).

## Cross-stage trends

### Friction patterns that recurred

- **Stub-driven `cargo-mutants` yields few viable mutants on `()`-returning
  wires.** Stage B's `enforcer.set_tier(tier)` (a unit-returning statement)
  produced 2 in-diff mutants, both unviable (`TestOutcome` is not `Default`), so
  the manual spot-check was primary — `cluster-pattern.md` §5 already sanctions
  this fallback, but the pattern (one-line wire → no viable mutant → manual
  analysis substitutes) recurs whenever a rung's new logic is a delegating
  statement rather than a value-returning branch. Stage E by contrast produced
  real viable mutants on the budget *branches* (warn/downshift/hard-stop +
  token accumulation), which the additive tests killed (`59bf723`, `bdf34be`).
- **The real-Anthropic behavioral leg is always the maintainer's IRL gate, never
  CI.** Every code rung's cluster closed on the **assembled CI test + the
  maintainer's IRL watch**, not tests-green — and each retro states this
  explicitly (rungs 1/3/4 IRL-observed; rung 5 engine-tested only, IRL deferred
  to the in-app budget surface). The `#[ignore]d` live evals (`skill_load_live`,
  `gap_detection_live`, `budget_live`) encode the behavior so it can't silently
  regress, but require a key — out of V's reach and run by the maintainer.

### Pattern-level wins

- **The execution-status ledger worked exactly as designed.** Each rung close
  produced an eval (E-01..E-05) that licensed a `paints → executes — observed`
  flip with an assembled-app cite — the structural fix for the M08.6 "Sound, 0🔴
  while the app was never run" escape. Rows 1, 3, 4 flipped on observed
  behavior; rows 2 (sub-agents), 5 (plans), 6 (hooks) stayed honestly `paints`.
- **The dispatch→feedback wire stayed singular.** V's Wire pass traced every
  rung-3/4/5 execution claim to a *production* call site in `drive_stream`
  (`agent_sdk.rs:619/642/681`), not a `#[cfg(test)]` harness — the "no
  production caller" condition that defined the M08.6 escape (and M08.V 🟡 #2)
  did not recur.
- **Strict v1.8 two-commit TDD held on all five rungs** with zero amend dances;
  V re-confirmed the empty test-diff invariant at Inventory.
- **Behavior over event.** Every rung asserts the side effect (file on disk;
  turn-count halt; suspend = exactly one provider turn; skill body in the turn-2
  config), never the emitted event alone — no gotcha-#66 / M08.6-#32-class
  event-only trap.

### Surprises across the parent milestone

- **Stage D pulled an architecture decision forward.** The maintainer's
  2026-06-01 direction ("a gap that dead-ends is not a runtime") turned the
  rung-4 suspend-only close into **ADR-0029** (gap resolve-and-resume, pulled
  into v0.1) — a scope expansion grounded in web best-practice (durable
  checkpoint-resume + idempotent rebuild + governed HITL re-authorization). The
  duplicate ADR-0029 created on-branch was deduped (`ac16004`) to defer to the
  canonical that landed on `main` via PR #119 — so the M08.7 branch carries no
  0029 file and the PR merge is conflict-free.
- **Budget grounded honestly at rung-5 entry.** The execution-status ledger
  refused to spec budget from assumption (rule 11); rung 5 grounded that budget
  was *tracked-but-not-enforced-at-the-loop* before authoring the eval, then
  wired the per-turn enforcement.

### Hard gate violations across the milestone

- **None.** All six hard gates (G1 do-not-commit-until-approved; G2 no
  Severity-5 friction; G3 no protocol drift; G4 stage actually completed; G5
  axis scores ≥3 in every row; G6 CI-parity) passed in every stage A–E + V. No
