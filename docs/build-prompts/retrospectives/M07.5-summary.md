## What this cycle closed

M07 Stage V's Wire pass produced one 🔴 finding
(`M07.V-retrospective.md` Finding #1): **`tier_gate` is defined but
never invoked in the import pipeline — a Novice "Reject" does not
reject.** `import_artifact_with` installed and hash-locked every
artifact before the tier-gate review could refuse it; the renderer's
Reject button only `delete`d a local store record — no Tauri command,
no backend rollback. Spec §8.security L4 ("every install is reviewed;
explicit Install click") and MVP §M7 ("Install/Reject") were violated.
The maintainer adjudicated that the fix exceeds scoped-D.fix scope — a
multi-crate import-lifecycle restructure — and waived it to this
dedicated fix-cycle via **ADR-0016**. M07.5 had to run **before M08
Stage A**, the milestone that introduces the production
artifact-load path, after which the gap would have a real blast radius.

The M07 closeout's simplify pass surfaced **CQ-M07-1**: the
import-fetch L1 capability gate (`EnforcerGate`) was tautological — it
granted itself the exact capability declaration it then checked. A
best-practice review found the deeper problem: the import pipeline
fetches a **user-supplied URL server-side** with no egress control —
the canonical Server-Side Request Forgery (SSRF) surface. The fix
cycle folded in a real SSRF-hardened egress gate, recorded as
**ADR-0018**.

Both findings are the gotcha #66 / #82 *tests-pass-but-contract-fails*
class: M07's unit + Stage-V suites were green, yet the **assembled
running app** did not honor the spec contract because Stage V verified
`tier_gate` in isolation against a constructed `ValidatedArtifact` —
it never asserted the *pipeline calls it*. The cycle's load-bearing
mandate (`CLAUDE.md` §6 v1.8): every regression test exercises the
**assembled running-app path**, and the phase-doc root cause is a
**falsifiable hypothesis the regression test must disprove**, not a
premise.

Outcome: **M07.V 🔴 #1 RESOLVED** (`51ef6eb` backend / `6178219`
renderer; ADR-0017 Accepted) and **CQ-M07-1 / SSRF RESOLVED**
(`e642843`; ADR-0018 Accepted), with three assembled-app regression
tests green in the full canonical CI gate suite as the durable
verification of record. The real-app manual GUI re-confirmation is
deferred-and-tracked to the post-M07.5 / M08 IRL pass (gotcha #23 — a
Tauri 2.x window cannot be driven or observed from the agent side; the
M06.5 between-milestone IRL precedent, maintainer-directed 2026-05-20).
**M08 Stage A is unblocked.**

## Cross-stage trends

### Friction patterns that recurred

- **Phase-doc illustrative code blocks are not gate-verified.** All
  three code stages hit the same class. A.fix — the `TierReview` /
  `ImportError` / `ImportOutcome` shapes tripped `result_large_err` /
  `large_enum_variant` (fixed by boxing the payload; deref coercion
  kept read sites unchanged). B.fix — the `egress.rs` doc comments
  tripped `too_long_first_doc_paragraph`, a red async stub tripped
  `unused_async`, a by-value param tripped `needless_pass_by_value`,
  an `.expect()` would have tripped `missing_panics_doc`. C.fix — a
  `recordImport` line exceeded prettier's print width. Each was
  absorbed within the self-correction budget and owned as a
  `CLAUDE.md` §12 technical decision. The standing candidate —
  gate-check phase-doc illustrative code/signatures/docs at
  authoring time — carries forward.
- **`warnings = "deny"` ⇒ large-red / small-impl for
  signature-changing stages.** A.fix + B.fix each produced a large red
  commit (the whole compiling scaffold — every new type, struct,
  command) and a small impl commit (the genuinely-deferred function
  bodies). A partial scaffold does not compile under the workspace
  `warnings=deny`, so it must all land in the red commit — which makes
  the strict-TDD invariant (impl touches no test files) *easier* to
  satisfy, not harder. A confirmed pattern across two stages, worth a
  `STAGE-PROMPT-PROTOCOL` note for future binary-crate `.fix` stages.

### Pattern-level wins

- **The assembled-app-regression mandate worked exactly as designed.**
  Each fix's regression test drives the assembled path —
  `import_artifact_with`'s real composition (A.fix + B.fix) and the
  real `ImportPanel` component (C.fix) — not the isolated primitive
  that already passed its unit test. This is the precise gotcha #66 /
  #82 gap M07's Stage V verified around: it tested `tier_gate` in
  isolation and never asserted the pipeline calls it.
- **Strict v1.8 two-commit TDD held cleanly on all three code
  stages.** Red standalone → impl with the test-diff-EMPTY invariant
  verified (the binary-crate `commands.rs` in-source-seam variant for
  A.fix + B.fix). The structural defense against silently rewriting
  tests to pass held.
- **Escalate-don't-pick discipline held.** Every phase-doc ambiguity —
  the `PendingImportState` manage placement (A.fix); the
  `cancel_pending_import` sync-vs-async signature (A.fix); the
  `NoHost` / `validate_egress` panic-free redesign (B.fix); the C.fix
  e2e invoke-mock infeasibility (gotcha #23) — was owned as a §12
  technical decision and surfaced in the red-phase surface for veto.
  None was picked silently. D.fix surfaced the manual-repro-vs-gotcha
  -#23 fork as a user-domain decision.

### Surprises

- C.fix's discriminated-union wire migration turned the **whole**
  renderer import suite red (10 vitest + 1 e2e), not just the 2 new
  closure tests — expected for a wire-shape `.fix`; the load-bearing
  right-reason signal is the *new* tests' assertions, not the failure
  count.
- D.fix's verification hit a self-inflicted contention class: running
  the Rust and frontend canonical gate suites concurrently produced
  two false-negative failures (`npm ci` wiping `node_modules`
  mid-`cargo test` broke the `xtask` `json2ts` drift check; the Rust
  llvm-cov compile CPU-starved the Playwright Vite dev server). An
  isolated sequential re-run of the identical CI commands cleared both
  — a local-environment lesson (run the suites sequentially), not a
  CI-parity break.
