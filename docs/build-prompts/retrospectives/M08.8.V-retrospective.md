## Findings

#### 🟡 #1 — assembled_execution: tier E2E is non-hermetic w.r.t. persisted `tier.json`

**Spec claim:** V.2 — "a Promoted run enforces Promoted … the tier display reads true across restart + a tier-change toast (#19/#20)"; close bar = the merge-blocking `e2e-tauri-driver` gate green (ADR-0021).
**Observed:** Full `npm run test:e2e:tauri` on this machine → **Spec Files: 8 passed, 2 failed, EXIT_CODE=1**. Both failures are the tier specs: `tier_display.e2e.ts:45` ("baseline tier button should read 'Promote' at Novice" — read "Demote") and `tier_enforcement.e2e.ts:33`/`:41` ("expected 'promoted' to include 'novice'"; "expected 'Demote' to equal 'Promote'"). Root cause: `%LOCALAPPDATA%/dev.aria-runtime.app/tier.json` = `{"tier":"promoted"}` from a prior session. After `cp`-resetting it to `{"tier":"novice"}` and re-running the two specs: **2 passed, EXIT_CODE=0** (3/3 assertions green). The product is correct; the tests assume a first-run-Novice baseline and rely on `tier_display`'s end-of-test cleanup-demote to restore state — so any earlier failure (or a machine already Promoted) **wedges the local gate**: `tier_display` fails at its baseline assertion → never reaches cleanup → leaves `tier.json` Promoted → every subsequent local run fails identically. CI is unaffected (fresh runner has no persisted `tier.json` → Novice default → green), so this is a **local merge-run wedge + a flake-recovery hole**, not a product regression.
**Expected (per spec):** the gate green on a maintainer's machine regardless of its persisted tier; tests independent of pre-existing `tier.json`.
**Trace / harness:** `npm run test:e2e:tauri` (real binary, tauri-driver) twice — as-found (red) and after Novice reset (green); `tier.json` read directly to confirm the root cause.
**Action:** small test-only fix (D.fix-class, ~15 min): a `before` hook in the tier specs (or a shared fixture) that writes `tier.json`=novice before the suite (and/or `after` that always restores), so the gate is hermetic. Zero-propagation — fix in M08.8 (do NOT carry to M08.9). Note for the maintainer: until fixed, reset `tier.json` to novice before running the full local gate, or the gate reads red on an already-Promoted machine.
