## [END] Special log (per `<retrospective_requirements>`)

**Pinned `test_framework` wire (from the shipped F1 code).** Command: `test_framework(app: AppHandle, framework_doc: Framework, task: String) -> Result<TestOutcome, CmdError>` (`src-tauri/src/commands.rs:1629`). `app` is Tauri-injected; the JS args are `frameworkDoc` + `task` (Tauri auto-converts the snake_case Rust `framework_doc` — the `importArtifact` precedent). The `testFramework` wrapper in `ipc.ts` calls `invoke<TestOutcome>('test_framework', { frameworkDoc: framework, task })`.

**Pinned `TestOutcome` serde shape (`crates/runtime-main/src/builder/tester.rs:47`).** `#[derive(Debug, Clone, serde::Serialize)]`, no `rename_all` ⇒ snake_case keys: `{ passed: bool, capability_failures: CapabilityFailure[], token_spend: TokenSpend, timing: Duration, vdr: Value, trace: AgentEvent[] }`. `CapabilityFailure = { agent_id, needed, reason }`; `TokenSpend = { input, output, total }`; `timing` is a Rust `Duration` ⇒ wire `{ secs, nanos }`. The `ipc.ts` TS mirror (`TestOutcome` / `CapabilityFailure` / `TokenSpend` / `WireDuration`) matches it field-for-field; `tsc --noEmit` clean.

**How the test-session graph is scoped.** A **Zustand store-factory**. `graphStore.ts`'s `create()` initializer was hoisted to a typed `StateCreator` const; `createGraphStore()` returns a fresh independent store from it; `useGraphStore` (the live runtime graph) is `createGraphStore()` and `builderStore.ts`'s `useTestGraphStore` is a second, independent `createGraphStore()`. The Tester reduces `outcome.trace` into `useTestGraphStore` only.

**The live `useGraphStore` singleton is untouched by a test run** — proven green by `running_a_test_does_not_write_to_the_live_useGraphStore_singleton` (vitest: after Run, `useGraphStore.getState().nodes` is `[]` while `useTestGraphStore.getState().nodes` is non-empty) and by the Playwright `closing_the_modal_discards_the_run_and_leaves_the_live_graph_untouched` (after close, `window.__graphStore.getState().nodes.length === 0`).

**Discard-on-close + the explicit Promote path.** `handleClose` drops the `TestOutcome` + task + error; `closeTester` clears `useTestGraphStore` — nothing is persisted (F1's backend already deleted the throwaway DB). The only persist path is the explicit "Promote to main session" button, which replays `outcome.trace` into the live `useGraphStore` then closes (spec Phase 9 "full graph available by promoting test session to main"). Green: `promote_writes_the_test_run_into_the_live_graph_store`.

**MVP §M8 criterion 5 (Playwright).** `tests/e2e/tester_modal.spec.ts` — 5 tests, all green: the Test button opens the modal; a task → Run renders the scoped graph pane (the real `<ReactFlow>` shows `agent-node-worker`); the VDR/token/pass-fail surfaces populate from the `TestOutcome`; a capability-violating framework surfaces a test-failure line with NO HITL modal; close discards the run and leaves the live graph untouched.

**Drift from the F1-shipped wire vs the F1.3.1 illustrative names.** The F1.3.1 param names (`framework_doc` / `task`) matched the shipped command — no drift there. Two drifts from the F2 phase-doc *pseudocode* (not the F1.3.1 names): (1) `timing` is serde's Duration `{ secs, nanos }`, not a ms number (F2.3.4); (2) the test session returns its trace in `TestOutcome.trace` and emits no live `agent_event`s (F2.3.3/F2.4.3). Both folded into the Decisions section.
