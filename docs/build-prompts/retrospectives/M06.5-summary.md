## Cross-stage trends

### Friction patterns that recurred

- **The phase doc's "verified root cause" was itself the gotcha #66
  trap one level up.** A.fix: the literal "`src-tauri/tests/`"
  instruction was structurally infeasible for a binary crate (no lib
  target → integration tests can't reach private composition fns) —
  resolved to an in-source `#[cfg(test)] mod tests` seam per CLAUDE.md
  §5. B.fix: the phase-doc root cause ("only a missing emission; no
  drone/IPC change; destination ready+tested") was *necessary but not
  sufficient* — a second condition (the `signals→sessions` FK + a
  drone-vs-SDK session-id mismatch) blocked persistence even with the
  emission wired; surfaced and maintainer-approved (Option A → ADR-0013).
  **Pattern:** "destination ready + tested in isolation" + a green
  isolated test is *itself* the gotcha #66 trap applied to the
  diagnosis, not just the code. Both were surfaced (not picked) and
  resolved with maintainer approval — the protocol held.

### Pattern-level wins

- **Strict v1.7 two-commit TDD held cleanly on both fix stages.** Red
  standalone → impl with the `'**/tests/**'`-EMPTY invariant verified
  (A.fix adapted it to a scoped diff for the binary-crate in-source
  seam; B.fix verified literal 0 lines). The structural defense against
  the agent silently rewriting tests to pass worked as designed.
- **Escalate-early discipline (CLAUDE.md §7).** B.fix escalated at
  iteration 1 of the green failure (the *diagnosis* was wrong, not the
  fix) rather than burning the 3-round budget — exactly the
  cross-stack-integration early-escalation rule.
- **Canonical gate ordering (fmt → clippy --fix → clippy -D → … →
  llvm-cov clean → gates) ran smoothly every stage.** Gotcha #81 (clean
  before measurement) + #56/#22 (drone-bin pre-build into
  `target/llvm-cov-target`) applied consistently; the CI-parity
  divergence is a prerequisite setup step, not a divergent measurement
  flag (cited at each stage's G-gate).

### Surprises

- C.fix received a **stale "Resume M06.5.B.fix" instruction** from an
  out-of-date orchestration view (citing a background-task ID not from
  the C.fix session, asking to re-commit already-landed B.fix work).
  Handled per CLAUDE.md §8/§12/§19-rule-7: the contradiction was
  surfaced with the authoritative build-machine git state rather than
  acted on — exactly the cross-session false-premise guard (the M04 PR
  #53 class). The session-end surface carries `git log --oneline
  main..HEAD` + the retro listing as a top-level item so any
  downstream orchestration session sees actual state.
