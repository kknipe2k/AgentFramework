## [END] Stage B special log (per `<retrospective_requirements>`)

**What ported cleanly from `tokens.css` / `workbench.css` vs needed adaptation.**
- *Clean (verbatim values):* the entire token `:root` — primitive (accent/surfaces/lines/text/status), semantic (`--kind-*` × color/text/bg, `--st-*`), type scale, spacing, radii, elevation, the `[data-accent|density|direction]` theming hooks, the `.t-*`/`.mono`/`.tnum` utilities, the instrument scrollbars. Ported value-for-value from `tokens.css`; token names kept DESIGN.md-identical (rename-free).
- *Adapted:* (1) the **legacy `--node-*` names** were remapped onto the tokens (e.g. `--node-bg: var(--surface-0)`) so the existing 2107-line node/panel CSS repaints to light without a per-selector rewrite — the key tractability decision. (2) The **Toast** dropped the mockup's × dismiss button to satisfy the `role=status`/no-interactive-children contract (auto-dismiss only). (3) The **Modal** added focus-trap + restore + `aria-labelledby` (`useId`) beyond the mockup's minimal version, per DESIGN.md's "every button responds" + WAI APG. (4) **Transport** ships the chrome (play/restart/scrub/speed) wired to the *existing* `invokeReplaySession` replay; scrub-to-seek is presentational in B (out of renderer-only scope). (5) **TesterModal** got the full-screen treatment via a CSS repaint of its existing classes, not a Modal-primitive swap (its unit tests pin the classes; the migration is test-bearing and deferred).

**The Plex-font offline outcome.** The `@import` of IBM Plex from Google Fonts was ported. Whether the Tauri WebView2 CSP fetches it is a real-app observation (the maintainer IRL / the `test:e2e:tauri` run will show it). The fallbacks (`system-ui`/`Segoe UI` for sans; `Cascadia Mono`/`Consolas` for mono) are in `--font-sans`/`--font-mono`, so a blocked fetch degrades to a system font, not a broken layout. **If the IRL shows the fallback font, the follow-up is to bundle the Plex `woff2` + a local `@font-face`** (noted, not done in B — it needs the font binaries committed).

**Stage-D divergences (shipped vs the mockup).** See the design-review note in the approval surface. Headline divergences, each a deliberate, surfaced call: (a) the Toast omits the mockup's × button (a11y); (b) the Tester is full-screen via CSS, not the Modal primitive (test-bearing migration deferred); (c) the transport scrubber is presentational in B (no scrub-to-seek). No *unintentional* divergences found in the unit/Playwright layer; the real-app conformance watch is the authoritative check.

**The close was the real-app watch (rule 11).** Per the prompt + ADR-0021, B closes on the maintainer launching the real Tauri app ("does it look like the mockup?") + `test:e2e:tauri` (`visual_foundation.e2e.ts`) — NOT Playwright-mock-green, NOT vitest-green. The vitest/Playwright greens are necessary, not sufficient.
