## [END] Decisions for the next stage (or next parent milestone)

### `CLAUDE.md` updates

- [ ] Candidate gotcha (§15) — **schema-pattern-bound fixture identifiers.** When authoring a `serde_json::from_value::<Framework>(json!({...}))` fixture in any test, every `name` / `id` / `model.id` field must match the schema regex (`^[a-z][a-z0-9_-]*$` for AgentId; ModelId is `^[a-z][a-z0-9_-]*$` per the schema). Dots, uppercase, and underscores-with-leading-letter all fail. **Symptom:** the first run panics at `serde_json::from_value` with `"doesn't match pattern \"^[a-z][a-z0-9_-]*$\""` instead of the right-reason test failure. **Mitigation:** copy a kebab-case identifier from an existing fixture in the same test file. Pairs with gotcha #66 (right-reason red) — a fixture-construction panic is NOT a right-reason red. Forward-applicable to every M08.6+ stage that touches framework JSON fixtures.
- [ ] Candidate gotcha (§15) — **walker-archetype-placement: free function in `framework_loader::capability_map`, not inline.** When a new SDK call site needs to walk `framework.agents[]` or `framework.tools[]` (e.g. extracting a derived property of an agent / a tool / a skill), the natural home is `crates/runtime-main/src/framework_loader/capability_map.rs` next to `inline_agents` / `parent_grants_for_agent` / `capabilities_for_tool` — NOT inline in the consumer (`agent_sdk.rs`, `tester.rs`, etc.).
