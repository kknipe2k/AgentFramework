No heartbeat / broadcast / out-of-band events on this channel. `await_validation_result` is single-shot (no skip-filter loop like drone's `await_event`). `next_response` borrow-not-move still applies because *consecutive* `validate` calls need the reader installed. Maintainer ack: confirm shape; otherwise Stage C2 events (e.g., isolation-installed notifications) would change the surface.
