## [LIVE] Byte-stability discovery (the HashMap-iteration trap)

The first impl pass passed three of four C.4 regressions but failed `save_load_save_is_byte_stable_on_a_modular_framework`. Diagnosis:

- `Framework` (`framework.rs:1263`) has three `HashMap<String, _>` fields: `hook_defs`, `mcp_aliases`, `per_mode_overrides`.
- `HashMap` iteration order is randomized per process by Rust's standard library (the SipHasher seed). Two saves in the same process produce different orderings.
- `serde_json::to_string_pretty(&Framework)` walks the struct fields, but each HashMap's `Serialize::serialize_map` iterates the underlying HashMap → randomized key order.
- The existing `save_load_save_cycle_is_byte_stable` test (`builder.rs:600`) passes only because `valid_framework_value()` has no `hook_defs` / `mcp_aliases` / `per_mode_overrides` entries (the fields' `skip_serializing_if = HashMap::is_empty` causes them to vanish entirely from the output). The byte-stability invariant was latently broken for any framework that exercised a HashMap field; aria's `hook_defs` (3 entries) exposes it.

The fix: canonicalize `framework.json` output through `serde_json::Value` before pretty-printing. The workspace's `serde_json` dependency does NOT enable the `preserve_order` feature (confirmed `Cargo.toml:22`), so `serde_json::Map` is backed by `BTreeMap` — every nested object's keys serialize alphabetically. One line of impl change (`let canonical = serde_json::to_value(&on_disk)?; let json = serde_json::to_string_pretty(&canonical)?;`); deterministic byte output independent of HashMap iteration.

Trade-off accepted: the output JSON's top-level fields are now alphabetically ordered (was struct-definition order via typify). Visible behavioral change for any save. The existing tests that asserted specific JSON fields parse correctly either way. The phase doc Key constraint "preserve the deterministic serialization (pretty-print, stable field order, trailing newline)" reads "stable field order" — alphabetical IS stable, and is the only way to make byte-stability work given the typify-generated HashMap fields.
