## [POST-IRL] Outcome — the maintainer IRL DISPROVED the slice (2026-06-08)

> **G4 did NOT close. The execution-status row was NOT flipped. M09.D is reopened as M09.D.fix.**

The maintainer real-app IRL ran the full walkthrough (author → role → `file_access.write` → attach MCP tool → Run, Promoted) and the slice **failed**: the agent ran **read-blind** and **wrote nothing**. Root cause — **the canvas-authored MCP tool was never injected into the model's tool list**, so the model never saw the tool, never called it, never received MCP data, and so had nothing to write. The on-disk side effect (a file within `out/` from real MCP data) did not occur, and the out-of-scope denial was therefore never even reached.

**What this exposes about the D assembled test (the honest correction).** `vertical_slice.e2e.ts` confirmed the *authoring composition* — the canvas-authored `framework_doc` is complete and RUNNABLE (`session_root_agent` rooted; `file_access.write` granted; the MCP tool in `allowed_tools` + `capabilities.tools_called`) — and the run *reached* `test_framework` and returned a verdict. But because no MCP server was installed in the CI/build env, the run executed **tool-free** (`commands.rs:1810`), so the test never exercised the path where the authored tool must be **surfaced to the model as a callable tool**. The D test's run leg asserted run-*reachability*, not tool-*injection* — and tool-injection is exactly where the assembled slice broke. structure ≠ behavior (rule 11): a complete, runnable doc whose tool never reaches the model is still a slice that does not run. The composition glue (`session_root_agent`) was necessary but **not sufficient**; the injection wire is the missing half.

**This is precisely the v1.8 assembled-mandate failure mode the IRL exists to catch** — the CI assembled test could not reach the tool-injection path without an installed MCP server, so only the maintainer's real-app run with a real server surfaced it. Recorded as a carry-forward and the M09.D.fix scope: surface the canvas-authored MCP tool into the model's tool list (mutation-blocking, since it is execution wiring — the tool-list build is a dispatch-deciding branch), plus the two DESIGN.md UI findings the IRL also surfaced (Tester resize; settings disclosure).

**Surfaced at:** 2026-06-08 (local). **M09.D.fix:** scoped and pending.
