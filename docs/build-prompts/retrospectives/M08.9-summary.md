## What M08.9 delivered

**The Tester stops lying.** A Novice run that was blocked from writing a
report used to render a green **PASS** with no file on disk — `fold_outcome`
folded an L2 `CapabilityViolation` but dropped an L4 `TierViolation` into
`_ => {}`, so a tier-blocked run returned `passed=true` and `TesterModal`
showed PASS (TD-047, IRL-confirmed). M08.9 makes the verdict **truthful**
(Pass / Fail / **Tier-limited**) and the run **drill-able** (verdict → each
tool call's input/result → raw), reusing the disclosure primitives that
already existed. This is the **trust prerequisite for M09's vertical slice**
(ADR-0031): the slice's run is only meaningful if a *blocked* run reports the
block instead of a green PASS.

- **A — Truthful verdict (closes TD-047).** `fold_outcome` folds
  `AgentEvent::TierViolation` (already in `outcome.trace`) into a new
  `TestOutcome.tier_blocks: Vec<TierBlock>` + a derived 3-state
  `verdict: TestVerdict` (`Pass` / `Fail` / `TierLimited`); `TesterModal`
  renders the verdict instead of the binary `passed` bool. `passed` is
  **unchanged** — a tier block is the user's tier setting, not a framework
  defect (ADR-0030). Hand-written `TestOutcome` Rust struct + its
  hand-written TS mirror stay in lockstep, producer-driven (gotcha #94); no
  schema change, no ADR.
- **B — Run drill-down.** `TesterModal` renders `outcome.trace` as verdict →
  per-tool-call input/result → raw, reusing the M08.8.A Output-rail payload
  surface (lifted to `src/lib/formatPayload.ts`) + the Show-raw disclosure
  (lifted to `src/components/RawDisclosure.tsx`, which `ValidationCard` now
  delegates to with DOM/testids preserved). Renderer-only; no new disclosure
  primitive, no backend/schema change.
- **V — Five-pass real-app verifier.** Fresh-context, bias-guarded; the 5th
  `assembled_execution` pass drove the real Tauri binary and **observed** the
  product correct (TIER-LIMITED ≠ PASS + no file; drill → payload → raw).
  Outcome **Sound but rough** — 1 🔴 (a test-harness defect, not a product
  defect) routed to D.fix.
- **D.fix — Close the Tester e2e teardown (V 🔴 #1).** The committed
  merge-blocking regressions never executed their substantive assertion
  (shared-session modal scrim intercepted the next case's click; crash on a
  key-present machine, skip on CI). A test-harness-only `afterEach`
  teardown restored them to executing green (12/12, exit 0). Graduated as
  gotcha #95.

## Cross-stage trends

### Friction patterns that recurred

- **Strict-TS `noUncheckedIndexedAccess` narrowing on new components only
  surfaces at green (B).** `steps[0]` / array-destructured `step` are
  possibly-undefined; narrow via `const step = steps[0]; if (step?.kind ===
  …)` rather than `const [step] = steps; expect(step.kind)…`. Same family as
  the M08.8.B note — a `<pre_red_mechanical_sweep>` (stub `tsc` on new
  components before the full run) for renderer stages would pre-empt it.
  Protocol-iteration candidate (not changed this milestone).
- **The §8-vs-strict-TDD two-local-commit tension (A + B).** §8 forbids
  `git commit` before approval, but the strict-TDD two-commit protocol +
  the approval surface both require two local commits to exist (to diff
  red..impl). Resolved every stage by keeping the commits local + unpushed;
  this is the de-facto protocol, noted for a possible one-line write-down.

### Pattern-level wins

- **"Fold it, don't detect it" / "reuse it, don't build it."** Both feature
  stages were small because the work was *composition*, not invention: A
  folded a signal already in `outcome.trace` (the fix was a ~3-line real-logic
  change — a match arm + the verdict); B reused two existing disclosure
  primitives via behavior-preserving lift-to-shared extractions. The phase-doc
  framing held exactly.
- **`wire_signature_audit` + `construction_reachability_check` paid off** on
  both stages — every pinned line (`event.rs:668`, `tester.rs:150/155`,
  `ipc.ts`, `TesterModal.tsx:39/43/51`) was load-bearing; B's assembled-modal
  wiring test *is* the reachability proof.
- **The verifier did its job.** V's 5th assembled-execution pass drove the
  real Tauri binary, separated *product-correct* from *harness-broken*, and
  scoped the single 🔴 precisely (test-only) — the bug class Stage V exists to
  catch (authored-but-never-run e2e). The fix path was pre-proven by V's
  throwaway spec, so D.fix was green on the first real-app run.

### Surprises across the parent milestone

- **The committed merge-blocking regressions had never executed their
  substantive assertions before V** — they crash on a key-present machine
  (modal-scrim intercept) and skip on CI (no `ANTHROPIC_API_KEY`). A spec can
  land "green" because the crash only fires when the real model key is present
  AND two cases share a session; CI's keyless skip masked it, and Stage B
  never ran the real app with a key. This is precisely the
  assembled-app-regression mandate's reason to exist; graduated as gotcha #95.
- **The "Output-rail payload component" was a module-private function, not a
  component (B).** The lift-to-shared was a function extraction
  (`formatPayload`), not a component reuse — the store-coupled `InspectorPanel`
  can't read `outcome.trace`. Recorded honestly; the reuse boundary was the
  pure formatter + the Show-raw disclosure, both lifted.

### Hard gate violations across the milestone

- **None.** Every stage held G1 (do-not-commit-until-approved — local
  unpushed commits, no PR), G2–G6. The one 🔴 was a Stage-V finding routed
  through the D.fix budget (1 of 2 iterations) and closed by observed
  real-app execution — the verifier protocol working as designed, not a hard
  gate failure.
