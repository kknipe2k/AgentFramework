- `framework_loader::walk` — `validate_framework` calls it for reference validation (`grep` in `validate.rs`: `use crate::framework_loader::walk;` + `walk(&framework)`).
- `framework_loader::capability_map::{parent_grants_for_agent, inline_agents}` — `framework_capability_summary` aggregates them (`grep` in `summary.rs`: `use crate::framework_loader::capability_map;` + `capability_map::inline_agents` / `capability_map::parent_grants_for_agent`).
- `capability::narrowing::narrow` — `framework_capability_summary` calls it per spawn edge (`grep` in `summary.rs`: `use crate::capability::narrowing::narrow;` + `narrow(&parent_caps, &child_declared_caps)`). The L2a intersection is **not** duplicated.
- `skills_lock::read` — `list_installed` is its **first production reader caller** (prior callers were tests only — `fan_out_grep` confirmed). The absent-lock case (`LockError::Io` of kind `NotFound`) maps to `Ok(vec![])`; every other lock error propagates.
- No Rust JSON-Schema validator exists — the schema-shape check is `serde_json::from_value::<Framework>` (the typify-generated type is the schema-as-source-of-truth, CLAUDE.md §14).

### The command-return-vs-spec-"events" §12 decision

Spec Phase 9 says validation "posts results back as events". `validate_framework` is shipped as a Tauri command returning `FrameworkValidationReport` (request/response), not a fire-and-forget event — the IPC has matured to synchronous command returns since the spec was written (M07's `import_artifact` / `request_tier_transition`). This is a §12-owned technical decision, recorded here; the **closeout gap-analysis owns the spec-phrasing refinement**. No schema, no event variant, no ADR.

### LoadedFramework byte-stable round-trip (ADR-0020 anticipation)

Confirmed — `save_load_save_cycle_is_byte_stable` passes: `to_string_pretty` + a trailing newline gives a deterministic `framework.json` serialization, and companion `.md` bodies are written verbatim, so a save→load→save cycle is byte-identical (MVP §M8 criterion 8). The `LoadedFramework` shape round-trips the canvas projection exactly — the ADR-0020 (Builder canvas↔`framework.json` state model, filed at Stage C) anticipation holds.
