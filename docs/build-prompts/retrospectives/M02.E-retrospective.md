The fix (sync on a visible-in-DOM indicator, re-query at click-time) is a generalizable Vitest+RTL idiom. | React Testing Library's documentation calls out that captured refs "may not reflect re-rendered DOM" — but the Vitest+RTL+userEvent stack hides this until tests run in a multi-file suite. Stage E hit this in 5 of 5 reruns with the prior pattern, 0 of 5 with the new pattern. | Promote to CLAUDE.md §15 / docs/gotchas.md: "When a Vitest+RTL test depends on a state transition triggered by an awaited promise, sync on a visible-in-DOM indicator (`findByLabelText` for an aria-label set in the success branch) BEFORE re-querying dependent UI.
