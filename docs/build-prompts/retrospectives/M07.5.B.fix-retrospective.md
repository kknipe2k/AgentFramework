## [SPECIAL LOG] Stage-required confirmations (per `<retrospective_requirements>`)

- **`classify_ip` covers every category — confirmed.** `classify_ip_rejects_every_internal_and_special_range` asserts rejection for 16 addresses: loopback (`127.0.0.1`), RFC-1918 (`10/8`, `172.16/12`, `192.168/16`), link-local (`169.254.169.254` — also the cloud-metadata address), CGNAT (`100.64.0.1`), unspecified (`0.0.0.0`), broadcast (`255.255.255.255`), IPv6 loopback (`::1`), IPv6 link-local (`fe80::1`), IPv6 ULA (`fc00::1`, `fd00::1`), IPv6 multicast (`ff02::1`), IPv6 unspecified (`::`), and **IPv4-mapped IPv6** (`::ffff:127.0.0.1`, `::ffff:10.0.0.1`). `classify_ip_accepts_public_addresses` covers `8.8.8.8`, `1.1.1.1`, a public IPv6. `classify_ip_unwraps_ipv4_mapped_ipv6_before_classifying` pins the `to_ipv4_mapped()` unwrap explicitly.
- **IP classification is HAND-ROLLED** (ADR-0018 §Decision default / Alternative C). No focused crate was adopted → **no `cargo deny` change**, no new dependency. The classifier composes the stable `std::net` predicates and hand-codes the three ranges `std` omits on the MSRV (CGNAT `100.64/10`, IPv6 ULA `fc00::/7`, IPv6 link-local `fe80::/10`); the exhaustive test matrix is the assurance.
- **The redirect-revalidation regression drives the assembled loop — confirmed.** `redirect_to_private_address_is_blocked` calls the real `import::fetch_with` with a `RedirectingFetcher` that 302s to `internal.example` (resolver-mapped to `10.0.0.1`); it asserts the error message contains `egress blocked` — distinguishing a *blocked-at-re-validation* outcome from mere loop-exhaustion, so a `fetch_with` that did not re-validate each hop would fail this test (gotcha #66 / #82).
- **DNS-pinning via `ClientBuilder::resolve` — confirmed.** `HttpFetcher::fetch_hop` builds a per-target client with `.resolve(&target.host, target.addr)`; the address is the one `validate_egress` resolved + classified, so the client cannot re-resolve elsewhere (the DNS-rebinding defense).
- **No new dependency — confirmed.** `reqwest::Url`, `tokio::net::lookup_host`, and `std::net` IP predicates are all already available; `Cargo.toml` / `Cargo.lock` are unchanged. `cargo audit` + `cargo deny check` are unaffected.
- **`host_of` + `NetworkGate` + `EnforcerGate` removed — confirmed.** The `host_of` URL parser, the `NetworkGate` trait, and the `EnforcerGate` struct + its impl are gone; a `grep` of `crates/runtime-main/src` + `src-tauri/src` finds only one occurrence — a historical reference in the `reject_to_import` doc comment.
