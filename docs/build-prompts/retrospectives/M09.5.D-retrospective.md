## [END] Special log (D.7 `<retrospective_requirements>`)

**The red proof (hang reproduced under guard).** The red commit `77c1fd4` (`provider_stall_suspend.rs`, `provider_resilience.rs`, `provider_retry_policy.rs`) failed-to-compile/failed-to-pass against `34325bf` because `providers::retry` + `providers::stream_guard` did not exist and `ProviderError::Overloaded` was unmodelled. Post-impl the same three files pass byte-identically (EMPTY-diff `git diff 77c1fd4..ace7bbf -- '**/tests/**'` is empty) ‚Äî the impl made the red go green without touching a test. The assembled hang itself: pre-D, the run loop awaited `stream.next()` with no deadline (`agent_sdk.rs:561`, deliberately untouched), so a stalled connection parked the session forever; `provider_stall_suspend.rs` reproduces it with a stalling fake and asserts the typed `Timeout` now surfaces into the existing `ProviderEvent::Error ‚Üí AgentError ‚Üí break` clean-suspend path.

**Where the adapter + retry landed vs the ¬ß6 exclusion (coverage honesty).** The timeout + retry **logic** lives in two NEW **covered** modules: `providers/retry.rs` (97.89 % line) and `providers/stream_guard.rs` (100 % line). `providers/anthropic.rs` ‚Äî the ¬ß6-**excluded** thin HTTP shell ‚Äî only *delegates*: `retry::execute(&RetryPolicy::default(), ‚Ä¶)` wraps each send, `retry::map_http_error(‚Ä¶)` maps non-2xx, and `stream_guard::with_idle_timeout(events, self.idle_timeout)` wraps the stream. No retry/timeout decision logic sits in the excluded file. runtime-main aggregate holds **96.15 % ‚â• 95**.

**The `Overloaded`-arm inventory (every match touched; no downstream match forced).** `ProviderError::Overloaded { retry_after_secs: Option<u64> }` added to the enum (`mod.rs`). Arms touched: `classify()` splits `Overloaded{Some}` (‚Üí `FlooredBackoff`, shares the `RateLimit` arm) from `Overloaded{None}` (‚Üí `Backoff`, shares the `Api{500|502|503}` arm); `map_http_error()` maps `status == 529 || body == overloaded_error` ‚Üí `Overloaded`. The variant carries its own `Display` (`overloaded_display_is_plain_language` pins it). No exhaustive `match` elsewhere in the workspace was forced to add an arm ‚Äî `ProviderError` is consumed via `Display`/`thiserror` propagation, not destructured downstream.

**Named constants + rationale.** `IDLE_TIMEOUT = 90 s` (per-event idle window; generous so model thinking gaps + API `ping` keep healthy streams well under it). `CONNECT_TIMEOUT = 10 s` (TCP/TLS connect ceiling on the reqwest builder; the idle timeout is the liveness instrument once streaming). `BACKOFF_BASE = 500 ms`, `BACKOFF_CAP = 8 s`, `RETRY_MAX_ATTEMPTS = 3` (first try + two retries) ‚Äî parity with the official Anthropic SDK defaults. All pinned by `retry_constants_are_pinned` + `idle_timeout_is_pinned_at_90s`.

**The clippy-driven `RetryClass` refactor.** `classify()` returns a `RetryClass` enum (`Never` / `Backoff` / `FlooredBackoff(u64)`) rather than an `Option<Option<u64>>`-style tuple ‚Äî a clippy/readability improvement that also makes the retry matrix legible. The matrix is pinned by `non_retryable_errors_fail_fast` (NEVER column: Auth, 400, 404) + `retryable_statuses_get_three_attempts` (RETRY column: 429, 500, 502, 503, 529) + `transient_failure_then_success_returns_ok`.

**Mutation survivors + disposition (BLOCKING gate ‚Äî critical module).** Scoped `--in-diff` to `git diff 34325bf..HEAD -- crates/runtime-main/src/providers/`; 53 mutants (14 anthropic.rs excluded-delegation, 39 retry.rs covered, 0 stream_guard ‚Äî see ambiguity note). **22 caught + 1 timeout-kill (`jitter %‚Üí+`) = 23 killed; 9 unviable (don't compile); 21 missed.** Every **required-kill** target died: `classify` `Overloaded{Some}`/`{None}` split + `500|502|503` (arm-deletes `137`/`141`), `RETRY_MAX_ATTEMPTS` bound (`179` `>=‚Üí<`), `map_http_error` `Overloaded` mapping (`211` `==`/`||` + `body_is_overloaded` true/false/`==`), 429 + 401|403 arms; stream_guard per-event-reset + fuse pinned by behavior test (0 generatable mutants). **All 21 survivors are on the jitter RNG-entropy or exact backoff magnitude** ‚Äî `with_seed`(seed), `seed_from_time`(entropy), the `| 1` odd-mix, `XorShift64::next`(PRNG `^=`/`<<`/`>>`), `jitter`(‚ÜíDefault / `+1` range), and `execute:184` (`attempt-1`‚Üí`attempt/1` exponent magnitude). **Justified inline per ¬ß5:** full-jitter's contract is "uniform in `[0, min(base¬∑2^n, cap)]`" ‚Äî `0` is a legitimate draw and the exact delay is deliberately randomized; the suite pins the contract that matters (never exceed the cap, honor the retry-after floor, correct attempt count, typed exhaustion) but cannot pin a non-zero/exact delay without asserting xorshift output, which is the ¬ß5 "tautologically restate the implementation" anti-pattern. No survivor sits on a retry **decision/dispatch** branch.
