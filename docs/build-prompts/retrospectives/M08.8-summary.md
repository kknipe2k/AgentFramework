## Cross-stage trends

### Friction patterns that recurred

- **"Recolored, not restyled" / "wired, not displayed" — the surface-completeness gap.** B ported the tokens but kept M03 spacing/borders (→ B.fix completed the restyle, surface-by-surface); C wired the tier enforcer but never seeded the tier *display* (→ C.fix). Twice the same shape: a stage delivered the structural half and a `.fix` stage closed the user-visible half. The lesson encoded forward: a UI stage's close bar is the **observed surface**, not the wired mechanism (rule 11 — enforcement-observed ≠ display-fixed; C.fix's whole premise).
- **Strict-TDD vs test-bearing migrations.** B deferred the `TesterModal`→`Modal` migration (its tests pinned `.tester-modal__*` classes — a test-bearing change forbidden in an impl commit); B.fix moved those test rewrites into its RED commit (TD-043 closed). B.fix2 hit the sharper variant: the fix required rewriting fabricated-fixture tests under `tests/`, handled as a two-commit RED(test-rewrite)→impl. The pattern: **a class/testid-pinning test makes a refactor test-bearing — schedule the test edit into a RED commit, never an impl commit.**
- **Thin-delegation wires produce only unviable cargo-mutants** (C: a `()`-returning `set_tier` wire → 2 unviable mutants → a hand-mutant proved the wire). Sibling to M08.7.B; `cluster-pattern.md` §5 covers it.

### Pattern-level wins

- **The real-app `tauri-driver` close bar worked exactly as designed.** Every built surface was observed in the running Tauri app (A 2/2, B 3/3, B.fix 5/5, B.fix2 6/6 incl. a live Anthropic run, C/C.fix tier specs on a clean baseline). The M08.6.V "Sound 0🔴 while the app was never run" escape **did not recur** — V's 5th `assembled_execution` pass drove the real binary for every surface.
- **Rename-free token/class discipline** kept a ~2,700-line stylesheet rebuild from breaking ~620 class-existence tests (B aliased legacy `--node-*` names onto the new tokens; B.fix changed property values, not class names). The single largest risk (a per-selector rewrite) was avoided by design.
- **Diagnose-first caught a fixture-fabrication bug a green suite was blind to** (B.fix2): the `replay.rs` translator read a field name that never existed; only the assembled-app run exposed it (the §5 tautology-test failure mode), and the fix deleted ~160 lines of hand-rolled reinvention in favor of one serde round-trip.

### Surprises across the parent milestone

- **B.fix2's fabricated-fixture disease** (gotcha #94) — unit + integration replay tests stayed green against invented signal shapes that produced zero events against real data. The rule-11 safety net (run the assembled app) was the only thing that caught it. Logged TD-045 (sweep for sibling sites).
- **The tier seam already existed** (`run_test_session_with_tier`) — C's TD-036 diagnosis ("painted, never wired") was exact; the gap was one delegation call in the Tauri command, not `tester.rs`.

### Hard gate violations across the milestone

- **None.** G1 (no-commit-without-approval) held every stage; no Severity-5 friction; no unaddressed protocol drift; strict v1.8 two-commit TDD held on every code stage (every `git diff <red>..<impl> -- '**/tests/**'` empty, V-confirmed at Inventory). The one process miss is the **absent A retrospective** — a §19 bookkeeping gap, not a hard-gate failure, closed by the backfill + V's Decision 4 standing check.
