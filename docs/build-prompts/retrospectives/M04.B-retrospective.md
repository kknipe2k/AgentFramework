| Coverage gates revealed cross-crate test coverage doesn't count toward per-package gates. The runtime-main integration tests (`plan_lifecycle.rs` + `plan_recovery.rs`) exercise the runtime-drone code via subprocess IPC, but `cargo llvm-cov --package runtime-drone` only counts tests living in runtime-drone. | Standard cargo coverage attribution: a test crate's coverage applies to the code it directly invokes from within the same package's test binary. Cross-crate integration tests via subprocess are seen by llvm-cov as "external execution" of a different binary. | When designing safety primitives that will be tested via cross-crate integration, plan to also write package-internal unit tests for the same logic. Clippy enforces a brevity discipline. | Add to docs/style.md or CLAUDE.md §9 conventions: "Module-level `//!` doc first paragraph: one sentence summary, max ~80 chars.
