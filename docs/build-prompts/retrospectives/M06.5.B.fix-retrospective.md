## [END] Special-log (prompt-mandated)

- **Where the persist call landed + why:** inside `AgentSdk::emit` (a private `persist_signal(&event)` called *before* the unchanged `event_tx.send`), not a sibling the run loop calls. `emit` is the single choke point every `AgentEvent` flows through (`agent_sdk.rs` AgentSpawned/pipeline/flush/MCP-dispatch sites), so persisting there makes the "signal count == emitted count" invariant hold **by construction** and guarantees no event class is missed. Persist-before-send so a slow/full renderer channel cannot starve the drone sink.
- **AgentEvent→signal field mapping derived + source:** from `crates/runtime-main/tests/recovery_lifecycle.rs:142-191` (the established `write_signal` call-site shape) + the `signals` columns in `crates/runtime-drone/migrations/000_initial.sql`. `signal_id` = fresh `Uuid::v4`; `session_id` = the run's shared `SessionId.as_string()` (same value `AgentSpawned` carries, `agent_sdk.rs:255`); `payload` = `serde_json::to_value(event)` (`AgentEvent` is `#[serde(tag="type")]`, matching recovery's `json!({"type":…,…})` shape); `event` = that `"type"` tag; `kind` = coarse `signals.type` (`tool_*`→`tool`, `decision*`→`decision`, `verify_*`→`verify`, else `agent` — consistent with recovery's kinds + `vdr::is_projection_eligible` keying on `decision|verify`); `context_type` = `"agent_loop"` (recovery's agent-loop context, `recovery_lifecycle.rs:170`). Not memory-authored.
- **token_usage:** `token_usage > 0` is **NOT** asserted (maintainer-approved scope decision). `signals > 0` **is** asserted, plus signal-count == emitted-count and run-session-id correctness. `token_usage`-never-populated is a distinct missing-projector defect recorded for C.fix / M07.A carry-forward.
- **Regression test drives the assembled path:** yes — `AgentSdk::run_agent` (the exact path `run_smoke_session_with` wraps with zero added logic, `commands.rs:165-172`) against a real drone subprocess, NOT a manual `client.write_signal()` (the `recovery_lifecycle.rs` Stage-V blind spot). A `runtime-main` integration test cannot depend on the `src-tauri` binary crate (dep cycle), so `AgentSdk::run_agent` is the faithful in-crate equivalent.
- **IPC-failure-tolerated posture:** `persist_signal` is best-effort — on `write_signal` `Err` it `tracing::warn!`s and returns (no `?`); the run never aborts on a transient drone-IPC failure. The unchanged `event_tx.send` keeps its `?` (a dropped renderer receiver is genuinely fatal). Pinned by `transient_signal_write_failure_does_not_abort_run` (drone killed mid-run → run still `Ok`, renderer sink intact).
- **Scope-expansion (load-bearing):** the true close of 🔴-2 required a *second* fix the phase doc did not diagnose — sharing the drone's seeded `SessionId` with the SDK (`src-tauri/src/{drone_lifecycle,main,commands}.rs`), parallel to 🔴-1/ADR-0012, no drone/IPC change. Maintainer-approved (Option A) before implementing.
