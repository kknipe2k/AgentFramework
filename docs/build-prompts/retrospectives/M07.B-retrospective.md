### special_log (required by the stage prompt)

- **Canonical-serialization choice:** `to_canonical_string` builds an explicit sorted `serde_json::Map` (`installed` keys `Vec::sort`ed) rather than relying on `serde_json`'s default `Map` ordering — so the output is byte-identical regardless of `HashMap` iteration order **and** of whether the `serde_json` `preserve_order` feature is enabled anywhere in the workspace. Pretty-printed (git-mergeable over compact) + trailing newline.
- **Cross-machine reproducibility test:** `canonical_serialization_is_byte_identical_regardless_of_install_order` — installs the same two artifacts in opposite order into two tempdirs and asserts the lock **bytes** are equal (not struct-equal) + keys alphabetically ordered. `content_hash_matches_known_sri_vector_for_empty_input` locks the cross-platform SRI digest constant.
- **Types generated, not hand-written:** `cargo xtask regenerate-types --check` clean; `schemas/skills-lock.v1.json` → `crates/runtime-core/src/generated/skills-lock.rs` (typify; `#[path]` snake_case module decl for the hyphenated file) + `event.v1.json` regen → `generated/event.rs` + `src/types/agent_event.ts`; `ArtifactHashMismatch` hand-mirrored into the curated `runtime_core::event::AgentEvent` per the BudgetWarn/McpRequestBlocked precedent.
