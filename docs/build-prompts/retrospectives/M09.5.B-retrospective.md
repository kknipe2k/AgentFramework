## [END] Special log (per the stage's `<special_log>`)

**Red-phase proof (the review finding reproduced).** Pre-fix, on this Windows machine: grant `{tmp}/out/**`, Write `{tmp}/out/../escape.txt` → `execute_builtin` returned `Ok({"bytes_written":18})` and `{tmp}/escape.txt` existed with content `ESCAPED-VIA-DOTDOT`. Junction variant: grant `{tmp}/granted/**`, junction `granted/jdir → outside`, Write `granted/jdir/escaped.txt` → `Ok`, file landed at `outside/escaped.txt` (`ESCAPED-VIA-JUNCTION`). Tests (a)/(e)/(f) failed red right-reason ("must emit CapabilityViolation" — none fired). Root cause confirmed: globset treats `..` as a literal component, and on Windows normalizes `\`→`/` in candidates, so both traversal forms *matched the grant glob* while the IO resolved them outside it.

**Symlink policy chosen + why.** Resolve-then-check, two layers: (1) lexical normalization feeds the enforcer check in the unchanged grant space (in-scope paths byte-identical — zero regression on authored grants); (2) the canonical resolved target (links followed) must sit under the canonicalized **literal anchor** of a matching grant — canonical-to-canonical, so `\\?\`, short names, and `/var → /private/var` never poison the comparison. Escaping links denied; in-grant links allowed; dangling links on a Write path refused (unverifiable target). The literal canonical-into-file_decl variant was rejected at plan approval on CI evidence (breaks green in-scope cases on macOS/Windows CI with unchanged grants).

**Rider-1 residual (verbatim in the policy comment).** Containment confines resolved targets to the grant's LITERAL ANCHOR — equal to the full scope for literal-prefix grants (`{dir}/**`, the v0.1 norm), coarser for metachar-bearing grants (under `{tmp}/*/out/**` a symlink can still move laterally anywhere under `{tmp}/`), and vacuous for a bare `**` grant by its own semantics. The claim is "resolved targets cannot escape the grant's literal anchor", never "symlinks cannot escape the glob". → One-line TD entry queued for Stage G closeout.

**Windows canonicalize friction.** `fs::canonicalize` yields `\\?\C:\…` verbatim paths — never fed to the glob (the glob only ever sees the lexical form); containment compares `Path::starts_with` component-wise on two same-API canonical paths; the IO runs on the canonical `PathBuf` (verbatim form is std::fs-safe). Win32's own lexical `..` collapse made case (d) green-pre-fix on Windows — the unix CI run is what proves the *normalizer* (not Win32) makes (d) pass post-fix.

**No in-scope E-02 case changed grants.** All three pre-existing E-02 cases and all 17 rung-1 cases pass post-fix with byte-identical grants; the diff over `**/tests/**` between red and impl is EMPTY.

**Mutation survivors + disposition.** Run 1 (16 mutants): 3 missed, 8 false-unviable (workspace `[lints]` deny built the stubs into compile errors — including the security-critical `confine_to_grant_anchor → Ok(())`). Dispositions: `delete !` in `normalize_lexical` → killed by the new irreducible-`..`/rooted-underflow units; guard-true in `canonicalize_for_write` → killed by the cfg(unix) file-as-directory unit on CI, locally unobservable on Windows (OS reports that shape as `NotFound`; inline justification at the guard); `+ → *` in `grant_literal_anchor` → provably equivalent (the slice is always `trim_end_matches('/')`-ed); refactored the `+ 1` away to remove the mutation surface. Final authoritative run with `--gitignore true --cap-lints true`: **14 mutants — 13 caught, 1 missed, 0 unviable**. The formerly-masked critical stubs (`confine_to_grant_anchor → Ok(())`, `execute_builtin → Ok(Default)`) are CAUGHT by the assembled E-02 suite. The lone survivor is the guard-true mutant at the `NotFound` walk-up guard — justified inline at the site (Windows reports the file-as-directory shape as `NotFound`, so the mutant is locally unobservable; both forms converge to the same `Op` at the IO there) and killed on unix CI by the cfg(unix) `write_canonicalize_propagates_a_file_as_directory_error` unit. The earlier `+ → *` equivalent mutant was removed by refactoring the redundant `+ 1` out of the anchor slice.
