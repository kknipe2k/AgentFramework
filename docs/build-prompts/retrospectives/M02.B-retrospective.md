- [x] **Did the trait surface feel right?** Yes. - [x] **Type names or signatures Stage C should change before SSE lands?** None proposed. Specifically:
  - `ProviderError::Sse(String)` is right for parse errors that don't fit `Unparseable` (e.g., framing issues vs body issues).
  - `ProviderError::RateLimit { retry_after_secs: u64 }` matches the Anthropic `retry-after` header convention (seconds, integer).
