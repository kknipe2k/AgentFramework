### Protocol additions (graduation candidates for `docs/gotchas.md` at M04 closeout)

- [ ] **Top-level `$ref` in JSON Schema breaks `json-schema-to-typescript`.** Pattern: "When authoring a schemas/*.v1.json file, the schema root must be a concrete type (`type: object`, etc.) — top-level `$ref` to a `$defs/...` entry triggers `parseNonLiteral` error in json-schema-to-typescript. typify (Rust side) supports both forms; TS codegen requires the concrete-type-at-root pattern. Inline the root definition's fields at the schema root and keep `$defs` for shared sub-types. Recurred M04.E (hitl.v1.json initial draft used top-level `$ref: '#/$defs/HitlPolicy'`)."
- [ ] **Module-doc intra-doc links to submodule items resolve at module-end scope.** Pattern: "Rustdoc resolves intra-doc links (e.g., `[`HitlPolicy`]`, `[`Desktop::with_dispatcher`]`, `[`super::terminal_bell`]`) at the END of the module's declaration scope, but the lint that fires on unresolved links is anchored at the doc-location (BEFORE the submodule / cross-module item is declared / re-exported). Use plain backticks for submodule + cross-module + cross-crate refs in module-level `//!` docs to avoid `cargo doc -D missing_docs` errors. Recurred M04.D `hooks/mod.rs` + M04.E `hitl/mod.rs` + `hitl/policy.rs` + `hitl/notifiers/{terminal_bell,sound,desktop}.rs`."
- [ ] **vitest fake timers + @testing-library/user-event don't compose cleanly.** Pattern: "When `vi.useFakeTimers()` is active, `userEvent.setup({ advanceTimers: vi.advanceTimersByTime }).click(...)` may time out at the test's default 5s budget — the user-event library's internal `wait` calls don't reliably progress fake timers across all environments. Use `fireEvent.click(...)` + `await waitFor(...)` directly for click-driven assertions when fake timers are needed elsewhere in the same test file; isolate fake timers to timer-specific tests (auto-dismiss, debounce, etc.). Recurred M04.E HITLToast initial draft."
- [ ] **Internal helper fns inside React component modules: re-export via `_testing` named export.** Pattern: "Component-internal pure-helper functions (decision filters, pure derivations, ARIA-string builders) often need direct unit-test coverage but should not pollute the public API. Re-export via `export const _testing = { helperFn };` at the component module's end; tests import the named export and assert on the helpers directly. The `_`-prefix marks the export as internal-test-only and is recognized by the eslint `no-unused-vars` allowed-unused-prefix rule. Pattern proven across M04.E HITLPanel/Modal/Toast components."
- [ ] **Tauri-plugin-notification cross-stack discipline upheld.** Pattern: "When integrating a Tauri 2.x plugin (notification, fs, dialog, etc.), the install path is uniformly: (a) `cargo add tauri-plugin-<name>` in `src-tauri/Cargo.toml` (use `"2"` as the version pin to track 2.x); (b) `npm install @tauri-apps/plugin-<name>` in `package.json`; (c) `tauri::Builder::default().plugin(tauri_plugin_<name>::init())` registered in main.rs setup hook; (d) add `<plugin>:default` permission to `src-tauri/capabilities/default.json`. WEBCHECK https://v2.tauri.app/plugin/<name>/ verbatim before authoring; the docs are version-stable. M04.E proved the gotcha #32 discipline works when the upstream is well-maintained."
