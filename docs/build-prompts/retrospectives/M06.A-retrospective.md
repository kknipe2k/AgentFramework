## [END] ADR-0009 closure summary

Per `<retrospective_requirements><special_log>`:

**Wire-up insertion points:**

- L1 wire-up insertion: `crates/runtime-main/src/sdk/event_pipeline.rs:115` (`ProviderEvent::ToolUse` branch in `next_event`) → `translate_tool_use` (line ~171). The function gates the dispatch through `enforcer.check` only when `self.enforcement` is `Some(EnforcementContext)`; pre-wired pipelines retain the M02 single-`ToolInvoked` behavior.
- L2a wire-up insertion: `crates/runtime-main/src/sdk/agent_sdk.rs:215` (post `AgentSpawned` emit, before streaming loop) → `spawn_framework_subagents` (line ~248). For each declared inline sub-agent, runs `narrow(parent_grants, proposed)`; on Ok emits `AgentSpawned` with `narrowed_from`; on Err emits `CapabilityViolation` and skips spawn.

**Four wire-up integration tests:**

- L1 (`crates/runtime-main/tests/sdk_capability_integration.rs`):
  1. `tool_dispatch_with_valid_grant_emits_capability_grant_and_tool_invoked`
  2. `tool_dispatch_missing_grant_emits_capability_violation_and_no_tool_invoked`
  3. `tool_dispatch_with_unknown_tool_emits_capability_violation`
  4. `tool_dispatch_twice_in_sequence_both_succeed_and_emit_separately` (gotcha #69 multi-call invariant)
- L2a (`crates/runtime-main/tests/sdk_narrowing_integration.rs`):
