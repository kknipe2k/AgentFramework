## Findings

#### 🔴 #1 — assembled_execution: `tests/e2e-tauri/tester_verdict.e2e.ts` + `tester_drilldown.e2e.ts` substantive cases never execute their assertion

**Spec claim:** The milestone close bar (V.2; A.6; B.6; the verification checklist) — "A + B closed on the **real Tauri app via `tauri-driver`** … a Novice run reads **TIER-LIMITED + no file** (not PASS); the drill-down expands to payload + raw" — encoded as the merge-blocking `e2e-tauri-driver` regression tests so it "can never silently regress" (cluster-pattern §4).

**Observed:** Ran `npm run test:e2e:tauri` (built the real app: vite + `cargo`/`tauri build`; `tauri-driver` + `msedgedriver`; WebView2 148). The suite **exited 1**. The two substantive cases failed identically:
- `tester_verdict.e2e.ts:67` and `tester_drilldown.e2e.ts:64`, both inside `openBuilderTester` at `builderTab.click()`:
  `WebDriverError: element click intercepted: Element <button data-testid="view-switch-builder"> is not clickable at point (394, 25). Other element would receive the click: <div class="modal-overlay" data-testid="modal-scrim">`
- Root cause: each spec's **first** `it` calls `openBuilderTester` (which calls `openTester()`), leaving the Tester `<Modal>` open. The **second** (substantive) `it` re-invokes `openBuilderTester`; the still-open modal's scrim intercepts the view-switch-tab click. No `afterEach`/teardown closes the modal between cases.
- `.env.local` supplies `ANTHROPIC_API_KEY` (2 vars) — confirmed by `smoke.e2e.ts`'s real-Anthropic case passing — so the `if (!hasAnthropicKey) this.skip()` guard did NOT fire; the substantive cases **ran and crashed** rather than skipping. On CI (no key) the same cases **skip**. Net: in neither environment does the substantive assertion execute.

**Expected (per spec):** The two specs drive the real app and assert TIER-LIMITED (not PASS) on a Novice Write run, and a tool-call drill to input/result + raw.

**Is the PRODUCT broken?** No — I drove the real app with a corrected throwaway spec (single `it` per case + `afterEach` closeTester; deleted after). Both assertions **passed on the running app**:
- A: a Novice Write run → verdict text == `TIER-LIMITED`, `tester-result--tier-limited` class, Promote affordance present (`verdict==tier_limited` ⇒ `passed && tier_blocks` non-empty ⇒ the model emitted Write, the L4 Novice gate blocked it, `TierViolation` folded). **TD-047 observably closed.**
- B: a Read run → drillable tool step → expanded input payload == the real `Cargo.toml` content; Show-raw reveals `tool_invoked`. **Drill-down observed.**

So the defect is **test-only** — but it defeats the milestone's permanent regression protection and leaves the merge-blocking suite red on a key-present machine.

**Trace / harness:** `npm run test:e2e:tauri` (full suite) → 10 passed, 2 failed, exit 1; corrected throwaway → 2 passing.
**Action:** Open D.fix iteration 1. Apply per-case teardown to both specs — an `afterEach` that calls `window.__builderStore.getState().closeTester()` (proven working in my throwaway), or restructure so the substantive case opens its own modal cleanly. Re-run `test:e2e:tauri` with a key and OBSERVE both substantive cases pass (suite exit 0). Cite finding #1 in D.fix's `<deliverable>`. Est. ~20 min.
