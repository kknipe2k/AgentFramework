## [END] Special-log entries (`<retrospective_requirements><special_log>`)

### §5a algorithm correctness — edge cases not obvious until implementation
- **First-`__` split (rule 4):** `split_once("__")` is exactly right — server names cannot contain `__` (McpServerName regex), so everything after the first `__` is the tool name verbatim, even if the tool itself contains `__` (test `resolve_canonical_splits_on_first_double_underscore_so_tool_may_contain_it`).
- **Disconnect can only REMOVE ambiguity, never add it.** `disconnect_server` re-evaluates symmetrically with `connect_server` but the newly-ambiguous delta is provably empty on removal. Kept the symmetric `ambiguous_short_names()` before/after diff so the re-evaluation site is one shape; the empty-on-disconnect property is asserted by the net-new test.
- **Rapid connect/disconnect/connect:** `NamespaceResolver` holds the connected snapshot; each connect/disconnect recomputes the ambiguous set from scratch (no incremental state to corrupt), so a connect→disconnect→connect cycle is order-independent and idempotent. No timing hazard because the resolver is `Arc<RwLock<>>`-guarded at the dispatcher and pure within the lock.
- **Alias vs canonical error distinction:** an `mcp_aliases` entry pointing at an unconnected canonical is `UnknownAlias` (a framework config error → `McpDispatchError::Config`), distinct from a short name that simply isn't found (`NotFound` → `None`, fall-through). This distinction is load-bearing for the dispatcher's "is this even an MCP tool" decision.

### Capability declaration shape (for Stage E renderer wiring + M06.V Wire-pass trace)
`mcp_tool_capability(server, tool)` → `CapabilityDeclaration { kind: Exec, resource: ResourceName("<server>__<tool>"), scope: Glob("<server>__<tool>"), side_effect_class: Irreversible }`. The enforcer's `subsumes` requires kind+side_effect_class+resource equality + `scope_contains` (glob==glob equality per M05.B). A framework grant built from the **same fn** for the same `(server, tool)` matches exactly. Implication for Stage E: an MCP tool call is `Exec`, which Novice tier forbids at L4 — the Settings/MCP UX must surface that MCP tool use needs Promoted tier (or the renderer routes the resulting `CapabilityViolation` + `McpRequestBlocked` through the existing capability-violation modal, which Stage D's graphStore branch already records under `capabilityViolations[agent_id]`).
