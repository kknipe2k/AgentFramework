### `CLAUDE.md` updates carrying forward (candidates from per-stage Decisions)

- [ ] **Candidate gotcha (§15)** — audit-first-author-second milestone
      pattern. "A milestone whose phase doc receives an authoring-time
      audit commit against the latest cumulative carry-forward
      (M08.6's `6936106` against the M08.5.5-summary lessons) converts
      the intake stage from 'absorb + author + triage' to 'verify +
      record'. The intake stage's friction collapses; the audit's
      hours of pre-work are paid forward by the milestone's remaining
      stages reading a pre-resolved carry-forward instead of
      re-resolving it. Forward-applicable to M09+." (Source: M08.6.A
      surprise event #1.)
- [ ] **Candidate gotcha (§15)** — HashMap-iteration breaks
      byte-stability of serialization. "When a typify-generated type
      carries `HashMap<K, V>` fields, direct
      `serde_json::to_string_pretty(&value)` produces non-deterministic
      key ordering because `HashMap`'s iteration is randomized.
      Byte-stable output requires `let canonical =
      serde_json::to_value(&value)?; serde_json::to_string_pretty(&canonical)?`
      — `serde_json::Map` is backed by `BTreeMap` (without the
      `preserve_order` feature, which this workspace does not enable),
      so every nested object's keys serialize alphabetically. This is
      the byte-stable write pattern for any `framework.json`-class
      deterministic-output file. Discovered M08.6.C `save_framework`
      re-split." (Source: M08.6.C surprise event + Decisions.)
- [ ] **Candidate gotcha (§15)** — additive-source dedup tests must
      assert source precedence, not just length. "When a stage adds a
      new source to a union (e.g., adding `framework` as a third
      source to a `built-in | installed | framework` palette union),
      the dedup test must assert the SURVIVING item's source
      discriminator — NOT just the union length. On `main` pre-impl
      the new source isn't read at all, so a bare length check passes
      tautologically (the installed-only or built-in-only result has
      the same length). The discriminator that fails right-reason is
      the data-source / data-origin attribute the new code is
      supposed to set." Forward-applicable to any future stage that
      folds in an additional source to an existing multi-source
      surface. (Source: M08.6.E surprise event #2.)
- [ ] **Candidate gotcha (§15) / refinement to #75** — `useShallow`
      is for derived selectors only. "`useBuilderStore((s) =>
      s.framework)` returns a single object reference — `s.framework`
      is the document itself, not a derived array/filter/map. Per
      gotcha #75, `useShallow` is the fix for derived selectors whose
      elements re-create per render; a bare-object selector returns
      the same reference across renders until `set()` swaps it, so
      reference equality is the right cache key. No `useShallow`
      import needed for bare-object selectors." (Source: M08.6.E
      [LIVE] useShallow consideration.)
- [ ] **Candidate gotcha (§15)** — `wdio` v9 `execute<R, A extends
      any[]>` is a dual-generic signature. "When typing the return
      value of `browser.execute(...)` in a wdio v9 e2e-tauri test, the
      signature is `execute<ReturnValue, InnerArguments extends
      any[]>(script, ...args): Promise<ReturnValue>`. A single-generic
      call (`execute<R>(...)`) fails TS2558 ('Expected 2 type
      arguments, but got 1'); the correct form for a string-script
      call with no args is `execute<R, []>(...)`. The single-generic
      form was a brief detour at M08.6.D." (Source: M08.6.D
      Decisions.)
- [ ] **Candidate gotcha (§15) or §6 update** — `e2e-tauri-driver` is
      a CI-only gate locally. "The `npm run test:e2e:tauri` job (wdio
      + tauri-driver against the built Tauri binary) requires `npx
      tauri build --no-bundle` + `cargo build --release -p
      runtime-drone -p runtime-sandbox --bins` + a `tauri-driver`
      launch — a multi-minute setup not run in the per-stage local
      verify_gates step. CI Linux + Windows are the structural
      verifiers per ADR-0021 + the `e2e-tauri-driver` job; the

      the verifier for it. The strict-TDD invariant (the test file
      identical in red + impl) holds either way." (Source: M08.6.D
      surprise event #2 + the M08.5 / M08.6 standing pattern.)
