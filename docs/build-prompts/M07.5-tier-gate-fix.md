## Background and Disposition

M07 Stage V's Wire pass produced one üî¥ finding
(`docs/build-prompts/retrospectives/M07.V-retrospective.md` Finding #1):
**`tier_gate` is defined but never invoked in the import pipeline; a
Novice "Reject" does not reject.** The maintainer adjudicated (commit
`8a861cd`) that the fix exceeds scoped-D.fix scope ‚Äî a multi-crate
import-lifecycle restructure ‚Äî and **waived it to this dedicated M07.5
fix-cycle via ADR-0016** (`docs/adr/0016-waiver-M07-tier-gate-deferral.md`,
Accepted). M07 merged (PR #87) with finding #1 open and carried forward.
**M07.5 MUST run before M08 Stage A** ‚Äî M08 introduces the production
artifact-load path, after which the gap would have a real blast radius.

The M07 closeout's simplify pass also surfaced **CQ-M07-1**: the
import-fetch L1 capability gate (`EnforcerGate`) is tautological. The
M07 gap-analysis promoted it to a üü° Fix-backlog item targeted at
M07.5. A best-practice review of that gate found the deeper problem:
the import pipeline fetches a **user-supplied URL server-side** and the
fetch path is unhardened against Server-Side Request Forgery (SSRF).
The right fix is not "make the tautology honest" ‚Äî it is a real
SSRF-hardened egress gate. ADR-0018 records that decision; M07.5 folds
it in as its own stage.

This cycle re-rails the work as **proper `.fix` stages with real red
gates**. A D.fix was attempted during M07 closeout ‚Äî red `518f6cf` +
WIP impl `bb71dbf`, preserved on the build-machine-local
`m07.5-salvage` branch ‚Äî but it was authored red+impl in one
undisciplined pass with no phase doc and no staged gate (the
`ORCHESTRATOR.md` ¬ß8 "Bad" case). M07.5 does **not** merge or
cherry-pick that WIP; it re-builds the design (ADR-0017 + ADR-0018) as
fresh, strict-TDD, gate-disciplined stages. The salvage branch is at
most a design *reference* the build machine may consult ‚Äî this phase
doc specifies the design fully and is the authoritative source.

**Verified root cause** (citations on `main` @ `ddf2a69`, authored
against read source ‚Äî not memory):

- **üî¥ #1 ‚Äî the lifecycle gap.** `tier_gate`
  (`crates/runtime-main/src/import/mod.rs:491`) returns
  `Err(ImportError::TierReviewRequired(TierReview { ‚Ä¶ }))` for
  `Tier::Novice`. It has **zero call sites**. `import_artifact_with`
  (`import/mod.rs:592-660`) skips it entirely ‚Äî the inline comment at
  `import/mod.rs:629-635` records the M07.C deviation ("the seam itself
  always installs + hash-locks ‚Ä¶ the review is a renderer confirm-gate")
  ‚Äî and unconditionally runs `reg.upsert(‚Ä¶)` (`import/mod.rs:636-639`,
  for `McpServer`) then `install_with` (`import/mod.rs:641`, the
  `skills.lock` write). The Tauri command
  (`src-tauri/src/commands.rs:1166-1222`) sets
  `review_required: matches!(tier, Tier::Novice)` (`commands.rs:1216`)
  **post-install**. The renderer's Reject button
  (`src/components/ImportPanel.tsx:154` ‚Üí
  `dismissImport(reviewItem.ref)`) calls `dismissImport`
  (`src/lib/graphStore.ts:1769-1777`), which only `delete`s the local
  store record ‚Äî no Tauri command, no backend rollback. There is **no
  `uninstall_artifact` / `skills_lock::remove`** anywhere. Net: a Novice
  "Reject" dismisses the panel record but leaves the `skills.lock`
  entry (and, for `mcp_server`, the MCP registry row). Spec
  ¬ß8.security L4 ("every install is reviewed; explicit Install click")
  and MVP ¬ßM7 ("Install/Reject") are violated.

- **CQ-M07-1 / SSRF ‚Äî the unhardened import-fetch egress.**
  `HttpFetcher::get` (`crates/runtime-main/src/import/fetch.rs:33-43`)
  is `reqwest::Client::new().get(url).send()` with default config ‚Äî it
  **auto-follows up to 10 HTTP redirects**, has no scheme restriction,
  no timeout, and reads the response body unbounded. `host_of`
  (`import/mod.rs:339`) is a hand-rolled URL host parser (a
  parser-confusion / IP-encoding bypass class). `EnforcerGate::check`
  (`src-tauri/src/commands.rs:1054-1077`) constructs a fresh
  `CapabilityEnforcer`, **grants itself the exact `decl` it then
  checks** (`commands.rs:1071-1075`), and returns the always-`Ok`
  result ‚Äî a tautology behind a doc comment claiming "default-deny +
  domain scope." Fetching a user-supplied URL server-side with no
  egress control is the canonical SSRF surface: a socially-engineered
  URL paste pivots the runtime into `localhost` / the LAN, DNS
  rebinding defeats a naive check, and redirect-following defeats an
  original-host-only check. ADR-0018 records the hardening decision.

ADR-0017 (`docs/adr/0017-import-validate-vs-commit-lifecycle.md`,
authored this cycle, `Proposed`) records the lifecycle design: the
validate / commit import-lifecycle split ‚Äî `import_artifact_with` runs
the pipeline **through** the tier-gate and **holds** a Novice import as
`Pending` (nothing installed, nothing locked); a renderer confirm runs
the held install half via a new `complete_import_artifact` command; a
renderer reject drops the held state via a new `cancel_pending_import`
command. ADR-0018 (`docs/adr/0018-import-fetch-ssrf-egress-hardening.md`,
authored this cycle, `Proposed`) records the egress design: an
SSRF-hardened import-fetch gate replacing the tautological
`EnforcerGate`.

The salvage branch's lifecycle design draft was numbered
`0016-import-validate-vs-commit-lifecycle.md` ‚Äî that number collides
with the merged waiver ADR-0016, so the canonical lifecycle design ADR
is **ADR-0017** (the "renumber 0016‚Üí0017" the waiver ¬ßDecision and
`ORCHESTRATOR.md` ¬ß9 call for; satisfied by authoring 0017 fresh and
the M07.5 branch carrying 0017, not the salvage 0016). The SSRF egress
hardening is new ‚Äî it was not on the salvage branch ‚Äî and is ADR-0018.

### Why four stages

The cycle has two independent backend concerns (the tier-gate
lifecycle and the fetch egress) plus a renderer rewire plus a closing
verification. The split is by **concern, language, and
compile-atomicity** ‚Äî the M07 Stage C / Stage E (backend / renderer)
precedent:

- **A.fix is the whole tier-gate-lifecycle Rust change and cannot be
  split further.** `import_artifact_with`'s return type changes
  (`Result<Installed, _>` ‚Üí `Result<ImportOutcome, _>`); that breaks
  its caller `commands.rs:1199` at compile time. runtime-main and
  `src-tauri` must land in **one** stage ‚Äî a stage that changed only
  runtime-main would leave the workspace non-compiling and unable to
  pass any gate (`ORCHESTRATOR.md` ¬ß2 ‚Äî split if large, but never into
  a non-compiling half).
- **B.fix is the import-fetch egress** ‚Äî a distinct code path (the
  `fetch_with` / `HttpFetcher` / egress-validation path, not the
  tier-gate control flow). It is a **security surface** warranting its
  own focused strict-TDD cycle with an exhaustive IP-classifier test
  matrix; folding it into A.fix would make A.fix oversized and muddy
  two unrelated red‚Üígreen diffs. Its only overlap with A.fix is a
  one-line `import_artifact_with` parameter swap (the `NetworkGate`
  gate ‚Üí a `Resolver`), cleanly owned by B.fix.
- **C.fix is the renderer**, a separate language with its own
  red‚Üígreen, pinning its IPC wrappers to A.fix's **shipped** command
  signatures via the v1.8 `<wire_signature_audit>` ‚Äî exactly M07's
  Stage E pinning against the shipped Stage C wire.
- **D.fix is the closing re-verification + reconciliation.** ADR-0016
  gates M08.A on üî¥ #1 re-tested green in the assembled app; that
  re-verification (now also covering the SSRF hardening) plus the
  `CHANGELOG` / summary / M08.A carry-forward reconciliation is real
  work that, folded into C.fix, would muddy C.fix's red‚Üígreen diff.

No Stage V: this is a D.fix-class remediation, not a milestone ‚Äî per
ADR-0008 the D.fix loop is itself the response to a V finding; it does
not recurse a V. M08's Stage V is the verification endpoint (ADR-0016
¬ßConsequences).

### Key constraints

- **Assembled-app regression mandate (load-bearing ‚Äî `CLAUDE.md` ¬ß6
  v1.8; gotcha #82).** Each fix's regression test MUST exercise the
  assembled running-app path, not the isolated component that already
  passes its unit test. The phase-doc root cause is a **falsifiable
  hypothesis the regression test must disprove**: A.fix's
  `reject_rolls_back_lock_and_registry` drives a Novice import through
  `import_artifact_with` and asserts no `skills.lock` entry + no
  registry row exist ‚Äî a test that **fails on `main` today** (today the
  pipeline installs). B.fix's `redirect_to_private_address_is_blocked`
  drives the real `fetch_with` redirect loop and asserts a redirect to
  a private address is rejected. C.fix's
  `reject_invokes_cancel_pending_import` drives the real `ImportPanel`
  component and asserts the Reject button fires the
  `cancel_pending_import` IPC.
- **Strict v1.8 two-commit TDD on A.fix + B.fix + C.fix.** Failing
  tests committed standalone (`test(M07.5.X.fix): ‚Ä¶`) and surfaced for
  red-phase approval **before** any implementation; the impl commit
  must **not** modify test files; the impl commit body states the
  verifiable invariant `git diff <red-sha>..<impl-sha> -- '<tests>'`
  is EMPTY. A.fix and B.fix span a lib crate (runtime-main, has
  `tests/`) **and** a binary crate (`src-tauri`, in-source
  `#[cfg(test)]`): the invariant is
  `git diff <red>..<impl> -- 'crates/runtime-main/tests/**'` EMPTY
  **and** the touched `src-tauri/src/*.rs` `#[cfg(test)]` block(s)
  byte-identical red‚Üíimpl (the binary-crate variant ‚Äî `CLAUDE.md` ¬ß6
  v1.8 / the M06.5.A.fix precedent ‚Äî proven by a scoped
  `git diff <red>..<impl> -- src-tauri/src/<file>.rs`). All test
  changes ‚Äî new tests **and** existing tests adapted to the new
  signatures ‚Äî land in the red commit. Net-new additive tests +
  mechanical fmt/clippy on test files go in a separate labelled
  follow-up commit.
- **Preserve ADR-0014 lock-on-first-install.** The `skills.lock` entry
  is still written exactly once, at the point of true install
  (`install_with`) ‚Äî for Promoted that is inline; for Novice that is
  `complete_import_with` on confirm. M07.5 changes *when* the install
  half runs, never the lock's content, schema, or SRI hash (no
  `schemas/skills-lock.v1.json` change ‚Üí no ¬ß14 schema ADR).
- **No framed-JSON IPC change.** `complete_import_artifact` /
  `cancel_pending_import` are **renderer‚Üîmain Tauri commands** ‚Äî the
  same IPC class ADR-0015 governs. The drone / sandbox framed-JSON IPC
  is untouched. ADR-0017 records the Tauri-command-pair addition.
- **SSRF egress: allow any public host, reject every internal range.**
  The import feature is "paste any GitHub-raw (or other public) URL"
  (MVP ¬ßM7) ‚Äî the control is an egress IP-classification blocklist, not
  a host allowlist (ADR-0018). HTTPS-only; `reqwest::Url` parsing;
  reject loopback / RFC-1918 / link-local / CGNAT / IPv6-ULA /
  IPv6-link-local / unspecified / multicast / IPv4-mapped-IPv6; DNS-pin
  to the validated address; re-validate every redirect hop; cap the
  body + timeouts.
- **Capability-enforcement surface (Hard Rule 8).** B.fix removes a
  capability-enforcement primitive (`NetworkGate` / `EnforcerGate`,
  the L1 import gate) and replaces it with a real egress gate. ADR-0018
  IS the surfaced plan. The egress *decision* logic must be a pure,
  exhaustively-tested seam (`classify_ip`, `check_url`); only the
  irreducible socket/DNS syscalls stay in the `import/fetch.rs`
  coverage holdout.
- **No new dependency.** B.fix uses `reqwest::Url` (re-exported by the
  existing `reqwest` dep), `tokio::net::lookup_host` (DNS), and
  `std::net` IP predicates ‚Äî all already available. A focused crate
  (e.g. `http-acl`) MAY be adopted for the IP-range checks **only** if
  it clears `cargo deny` (Hard Rule 6); the default is the hand-rolled
  classifier with an exhaustive test matrix (ADR-0018 Alternative C).
- **No scope creep.** Touch only what üî¥ #1 and the import-fetch
  egress hardening require. No üü°-#2/#3/#4/#5 work, no Reinstall, no
  TD-014..018 cleanup, no protocol edits.
- **No Co-Authored-By trailers.** Commit messages end with the
  session-URL footer only.
- **Windows is the v0.1 target; CI runs all three OSes.** The
  assembled regression harnesses reuse the existing
  `crates/runtime-main/tests/import_pipeline_integration.rs`
  injected-seam pattern (cross-platform in CI; no live network ‚Äî Hard
  Rule 4 / `CLAUDE.md` ¬ß10).
- **Capability adherence (`CLAUDE.md` ¬ß10).** Regression tests use the
  injected `Fetcher` / `Sandbox` / `McpRegistry` / `Resolver` fakes; no
  test reaches the live network. No `--features integration` in the
  normal gate flow.

## Stage A.fix ‚Äî üî¥ #1 backend: import validate/commit lifecycle split

### A.1 Problem Statement

The import pipeline installs and hash-locks every artifact before the
Novice tier-gate review can refuse it; the review is cosmetic and the
"Reject" click has no backend effect.

Verified (citations on `main` @ `ddf2a69`):

- `crates/runtime-main/src/import/mod.rs:491` ‚Äî `tier_gate(a, tier,
  report)` returns `Err(ImportError::TierReviewRequired(TierReview { ‚Ä¶ }))`
  for `Tier::Novice`, `Ok(())` for `Tier::Promoted`. **Zero callers.**
- `import/mod.rs:592-660` ‚Äî `import_artifact_with` runs fetch ‚Üí validate
  ‚Üí ¬ß15c ‚Üí L3, then (skipping `tier_gate`) `reg.upsert` for `McpServer`
  (`:636-639`) and `install_with` (`:641`, the unconditional
  `skills.lock` write), returning `Ok(Installed { ‚Ä¶ })`. The
  `:629-635` comment records the M07.C deviation rationale.
- `install_with` (`import/mod.rs:556-580`) writes the `skills.lock`
  entry via `skills_lock::write_entry`.

Per ADR-0017 (Option A ‚Äî install-after-confirm): `import_artifact_with`
runs the pipeline **through** `tier_gate`; a passed gate (Promoted)
installs inline; a `TierReviewRequired` (Novice) **holds** the import
as a `PendingImport` and installs nothing. A new
`complete_import_with` runs the held install half on the renderer's
confirm; a reject simply drops the held state ‚Äî because the install
half never ran, there is nothing to roll back.

Why üî¥ for M08: M08 introduces the production artifact-load path; an
artifact a Novice rejected but that stayed locked/registered would
become reachable. M07.5 closes the gap first (ADR-0016 ¬ßConsequences).

Stage V could not see this: it tested `tier_gate` in isolation against
a constructed `ValidatedArtifact`. It never asserted the **pipeline
calls it** such that a Novice import produces no lock. That assertion
is this stage's regression test.

### A.3 Detailed Changes

#### A.3.1 The validate/commit lifecycle split ‚Äî `crates/runtime-main/src/import/mod.rs`

`tier_gate` already produces the right outcome (`import/mod.rs:491-500`);
the fix is to **call** it and branch the pipeline on the result. The
review screen renders a share-provenance trust line, so `TierReview`
gains `share_provenance` (today it is on `ArtifactMeta` only) ‚Äî making
`TierReview` the complete review primitive the `Pending` arm carries:

```rust
/// The capability disclosure the Stage E review screen renders for a
/// Novice import (spec ¬ßM7).
#[derive(Debug, Clone, PartialEq)]
pub struct TierReview {
    /// Human-readable declared-capability summary for the disclosure.
    pub capabilities: Vec<String>,
    /// The L3 report shown alongside the disclosure.
    pub l3_report: L3Report,
    /// Secrets to provision before first run (spec ¬ß15d).
    pub requires_secrets: Vec<String>,
    /// The ADR-0005 `share_provenance` trust block (when exported) ‚Äî
    /// the review modal renders it as a trust line. `None` when the
    /// artifact was never exported through the trust chain.
    pub share_provenance: Option<Value>,
}
```

> **Note** `Eq` is dropped from `TierReview` ‚Äî `serde_json::Value`
> is `PartialEq` but not `Eq`. `PartialEq` (used by `assert_eq!`) is
> preserved. This mirrors the `Installed` struct's existing
> `PartialEq`-only derive (`import/mod.rs:192` doc comment).

A `PendingImport` carries a held Novice import ‚Äî everything
`complete_import_with` needs to finish, the pipeline having already run
fetch / validate / ¬ß15c / L3:

```rust
/// A Novice import held at the tier-gate review (M07.5 / ADR-0017 ‚Äî
/// the install-after-confirm split). The pipeline has already run
/// fetch / validate / ¬ß15c / L3; the only work left is the install
/// half ‚Äî MCP-registry upsert (`mcp_server` only) + install + lock ‚Äî
/// which `complete_import_with` runs on the renderer's confirm.

    art: ValidatedArtifact,
    src: ImportSource,
    report: L3Report,
    tier: Tier,

impl PendingImport {

    /// The renderer keys its review record by this ‚Äî it is stable
    /// across the pending ‚Üí installed transition.
```

`import_artifact_with` returns a discriminated outcome instead of a
bare `Installed`:

```rust
/// The outcome of [`import_artifact_with`] (M07.5 / ADR-0017 ‚Äî the
/// validate/commit split that closes M07.V üî¥ #1).
///
/// - `Installed` ‚Äî the tier-gate passed (Promoted, L4 auto-accept);
///   the artifact is installed + hash-locked inline.
/// - `Pending` ‚Äî the tier-gate requires a Novice capability-disclosure
///   review (spec ¬ß8.security L4); NOTHING is installed or locked. The
///   renderer shows the review screen; [`complete_import_with`]
///   finishes the install on confirm, or the pending state is dropped
///   on reject (nothing to roll back ‚Äî the install half never ran).
#[derive(Debug)]
pub enum ImportOutcome {
    /// Installed + hash-locked (Promoted / L4 auto-accept).
    Installed(Installed),
    /// Held for a Novice review ‚Äî no install, no lock (the üî¥ #1 fix).
    Pending {
        /// The capability disclosure the review screen renders.
        review: TierReview,
        /// The in-flight import [`complete_import_with`] finishes.
        pending: PendingImport,
    },
}
```

The install half is extracted into a private `commit_import` so the
inline (Promoted) path and the deferred (`complete_import_with`) path
run **byte-identical** install logic:

```rust
/// The install half of the import pipeline ‚Äî MCP-registry upsert
/// (`mcp_server` only) + install + `skills.lock` write. Runs ONLY past
/// a passed tier-gate: inline for Promoted, or via
/// [`complete_import_with`] on a Novice confirm. It NEVER runs for an
/// unconfirmed Novice review ‚Äî that is the M07.V üî¥ #1 fix.
///
/// The MCP-registry upsert is ordered AFTER the `skills.lock` write:
/// the lock is the ADR-0014 integrity source of truth, and the two
/// stores are not transactionally linked ‚Äî if the upsert fails after
/// the lock is written, the artifact is locked-but-not-registered (a
/// re-import reconciles it), which is the more recoverable partial
/// state than a registry row with no lock entry.
fn commit_import(
    art: &ValidatedArtifact,
    src: &ImportSource,
    report: &L3Report,
    tier: Tier,
    reg: &dyn McpRegistry,
    lock: &Path,
    clock: &dyn Clock,
) -> Result<Installed, ImportError> {
    install_with(art, src, report, tier, lock, clock)?;
    if matches!(art.kind, ArtifactKind::McpServer) {
        reg.upsert(&mcp_import_of(art)).map_err(ImportError::Registry)?;
    }
    Ok(Installed {
        lock_key: art.name_at_version(),
        report: report.clone(),
        requires_secrets: art.meta.requires_secrets.clone(),
        capabilities: capability_summary(&art.raw),
        share_provenance: art.meta.share_provenance.clone(),
    })
}
```

> **Ordering note** ‚Äî the M07.C code ran `reg.upsert` *before*
> `install_with`. `commit_import` flips the order (lock first) per the
> doc comment above. This is a deliberate, small correctness
> improvement folded in because M07.5 is already extracting this
> function; full cross-store atomicity is out of scope (ADR-0017
> ¬ßConsequences).

`import_artifact_with` now **calls `tier_gate`** between L3 and the
install half ‚Äî the one line M07.C removed:

```rust
#[allow(clippy::too_many_arguments)]
pub async fn import_artifact_with(
    src: ImportSource,
    kind: ArtifactKind,
    tier: Tier,
    host_os: &str,
    lock: &Path,
    gate: &dyn NetworkGate,
    get: &dyn Fetcher,
    sb: &dyn Sandbox,
    reg: &dyn McpRegistry,
    clock: &dyn Clock,
) -> Result<ImportOutcome, ImportError> {
    let bytes = fetch_with(&src, gate, get).await?;
    let art = validate(kind, &bytes)?;

    // ¬ß15c ‚Äî BLOCKING, and BEFORE the expensive L3 (cheap reject).
    if !art.meta.compatible_os.iter().any(|o| o == host_os) {
        return Err(ImportError::OsMismatch {
            artifact: art.meta.compatible_os.clone(),
            host: host_os.to_string(),
        });
    }

    let code = String::from_utf8_lossy(&art.bytes);
    let reasons = sb
        .validate(&code)
        .await
        .map_err(|e| ImportError::L3(vec![e]))?;
    if !reasons.is_empty() {
        return Err(ImportError::L3(reasons));
    }
    let report = L3Report {
        report_id: uuid::Uuid::new_v4().to_string(),
        passed: true,
        reasons: Vec::new(),
    };

    // L4 tier-gate ‚Äî the M07.5 install-after-confirm gate (closes
    // üî¥ #1). Promoted ‚Üí Ok ‚Üí install inline. Novice ‚Üí
    // TierReviewRequired ‚Üí HOLD: nothing is upserted or locked until
    // `complete_import_with` runs on the renderer's confirm.
    match tier_gate(&art, tier, &report) {
        Ok(()) => {
            let installed =
                commit_import(&art, &src, &report, tier, reg, lock, clock)?;
            Ok(ImportOutcome::Installed(installed))
        }
        Err(ImportError::TierReviewRequired(review)) => {
            Ok(ImportOutcome::Pending {
                review,
                pending: PendingImport { art, src, report, tier },
            })
        }
        // `tier_gate` yields only `Ok` or `TierReviewRequired`; this
        // arm is unreachable in practice but keeps the match total
        // without a panic.
        Err(other) => Err(other),
    }
}
```

`PendingImportState` is Tauri-managed state holding held imports
between `import_artifact` returning `Pending` and the renderer's
`complete_`/`cancel_` call:

```rust
/// Tauri-managed state ‚Äî Novice imports awaiting a tier-gate review
/// confirmation (M07.5 / ADR-0017). Keyed by `pending_review_id`. An
/// entry lives only between `import_artifact` returning `Pending` and
/// the renderer's `complete_import_artifact` / `cancel_pending_import`.
/// A plain `std::sync::Mutex` ‚Äî the critical sections are a map insert
/// / remove with no `.await` held across the lock.
///
/// `MAX_PENDING` bounds the map: a renderer that creates `Pending`
/// records without ever resolving them cannot grow it without limit
/// (each entry holds the fetched artifact bytes). v0.1 is
/// single-session and reviews one import at a time, so the bound is
/// generous defense-in-depth, not a normal-path limit.
#[derive(Default)]
pub struct PendingImportState(
    std::sync::Mutex<std::collections::HashMap<String, import::PendingImport>>,
);

const MAX_PENDING: usize = 16;

impl PendingImportState {
    /// Insert a held import. Returns `Err` if the map is at
    /// `MAX_PENDING` (the caller maps it to a `CmdError`).
    fn insert(&self, id: String, pending: import::PendingImport) -> Result<(), String> {
        let mut map = self.0.lock().expect("PendingImportState mutex poisoned");
        if map.len() >= MAX_PENDING {
            return Err(format!("too many pending imports (max {MAX_PENDING})"));
        }
        map.insert(id, pending);
        Ok(())
    }

    /// Remove and return the held import for `id`. `None` for an
    /// unknown id ‚Äî a `complete`/`cancel` double-fire ‚Äî handled
    /// idempotently by the callers.
    fn take(&self, id: &str) -> Option<import::PendingImport> {
        self.0
            .lock()
            .expect("PendingImportState mutex poisoned")
            .remove(id)
    }
}
```

`import_artifact` maps the runtime-main `import::ImportOutcome` to the
wire enum ‚Äî and, for `Pending`, mints the `pending_review_id` and
stashes the held import:

```rust
#[tauri::command]
pub async fn import_artifact(
    app: AppHandle,
    source_kind: String,
    location: String,
    artifact_kind: String,
    sandbox: tauri::State<'_, Arc<SandboxClient>>,
    registry: tauri::State<'_, Arc<Registry>>,
    tier_state: tauri::State<'_, CurrentTierState>,
    pending: tauri::State<'_, PendingImportState>,
) -> Result<ImportOutcome, CmdError> {
    let dir = app
        .path()
        .app_local_data_dir()
        .map_err(|e| CmdError::internal(format!("app_local_data_dir: {e}")))?;
    let lock = dir.join("skills.lock");

    let src = match source_kind.as_str() {
        "url" => ImportSource::Url(location),
        "file" => ImportSource::File(location.into()),
        other => return Err(CmdError::internal(format!("unknown source_kind: {other}"))),
    };
    let kind = match artifact_kind.as_str() {
        "skill" => ArtifactKind::Skill,
        "tool" => ArtifactKind::Tool,
        "agent" => ArtifactKind::Agent,
        "mcp_server" => ArtifactKind::McpServer,
        other => return Err(CmdError::internal(format!("unknown artifact_kind: {other}"))),
    };
    let tier = *tier_state.lock().await;

    let outcome = import::import_artifact_with(
        src,
        kind,
        tier,
        std::env::consts::OS,
        &lock,
        &EnforcerGate,
        &HttpFetcher::new(),
        &SandboxAdapter(Arc::clone(sandbox.inner())),
        &RegistryAdapter(Arc::clone(registry.inner())),
        &SystemClock,
    )
    .await
    .map_err(|e| import_err_to_cmd(&e))?;

    Ok(match outcome {
        import::ImportOutcome::Installed(installed) => ImportOutcome::Installed {
            lock_key: installed.lock_key,
            capabilities: installed.capabilities,
            l3_report: installed.report,
            requires_secrets: installed.requires_secrets,
            share_provenance: installed.share_provenance,
        },
        import::ImportOutcome::Pending { review, pending: held } => {
            let pending_review_id = uuid::Uuid::new_v4().to_string();
            let lock_key = held.lock_key();
            pending
                .insert(pending_review_id.clone(), held)
                .map_err(CmdError::internal)?;
            ImportOutcome::Pending {
                pending_review_id,
                lock_key,
                capabilities: review.capabilities,
                l3_report: review.l3_report,
                requires_secrets: review.requires_secrets,
                share_provenance: review.share_provenance,
            }
        }
    })
}
```

```rust
/// Finish a Novice import the renderer confirmed at the tier-gate
/// review (M07.5 / ADR-0017). Takes the held `PendingImport` out of
/// `PendingImportState`, runs the install half
/// (`import::complete_import_with`), and returns the terminal
/// `Installed` outcome.
///
/// # Errors
///
/// [`CmdError::Internal`] when `pending_review_id` is unknown (already
/// completed / cancelled) or the install half fails.
#[tauri::command]
pub async fn complete_import_artifact(
    app: AppHandle,
    pending_review_id: String,
    registry: tauri::State<'_, Arc<Registry>>,
    pending: tauri::State<'_, PendingImportState>,
) -> Result<ImportOutcome, CmdError> {
    let dir = app
        .path()
        .app_local_data_dir()
        .map_err(|e| CmdError::internal(format!("app_local_data_dir: {e}")))?;
    let lock = dir.join("skills.lock");

    let held = pending.take(&pending_review_id).ok_or_else(|| {
        CmdError::internal(format!("unknown pending import: {pending_review_id}"))
    })?;
    let installed = import::complete_import_with(
        &held,
        &RegistryAdapter(Arc::clone(registry.inner())),
        &lock,
        &SystemClock,
    )
    .map_err(|e| import_err_to_cmd(&e))?;

    Ok(ImportOutcome::Installed {
        lock_key: installed.lock_key,
        capabilities: installed.capabilities,
        l3_report: installed.report,
        requires_secrets: installed.requires_secrets,
        share_provenance: installed.share_provenance,
    })
}
```

## Stage B.fix ‚Äî import-fetch SSRF egress hardening (CQ-M07-1)

### B.1 Problem Statement

The import pipeline fetches a user-supplied URL server-side with no
egress control ‚Äî the canonical Server-Side Request Forgery surface.

Verified (citations on `main` @ `ddf2a69`):

- `crates/runtime-main/src/import/fetch.rs:33-43` ‚Äî `HttpFetcher::get`
  is `reqwest::Client::new().get(url).send()`: default config
  **auto-follows up to 10 redirects**, no scheme restriction, no
  timeout, unbounded `resp.bytes()` body read.
- `import/mod.rs:339` ‚Äî `host_of` is a hand-rolled URL host parser
  (parser-confusion / IP-encoding bypass class).
- `import/mod.rs:364-378` ‚Äî `fetch_with` calls `host_of` ‚Üí
  `gate.check(host)` ‚Üí `get.get(u)`.
- `src-tauri/src/commands.rs:1054-1077` ‚Äî `EnforcerGate::check`
  constructs a fresh `CapabilityEnforcer`, grants itself the exact
  `decl` it then checks (`:1071-1075`), and returns the always-`Ok`
  result ‚Äî a tautology (CQ-M07-1).

Per ADR-0018: replace the tautological gate with a real SSRF-hardened
egress gate ‚Äî HTTPS-only; parse with `reqwest::Url`; resolve and
classify every IP (reject loopback / private / link-local / CGNAT /
IPv6-ULA / IPv6-link-local / unspecified / multicast / IPv4-mapped);
DNS-pin the connection to the validated address; re-validate every
redirect hop; cap the body + timeouts. The decision logic is a pure,
exhaustively-tested `import::egress` module; only the socket / DNS
syscalls stay in the `import/fetch.rs` holdout.

Why this is the right fix, not the "honest no-op": a server-side fetch
of a user-supplied URL with no egress control lets a socially-
engineered URL paste pivot the runtime into `localhost` / the LAN; DNS
rebinding defeats a naive validate-then-fetch check; redirect-following
defeats an original-host-only check. The `EnforcerGate` doc comment
already *claims* "default-deny + domain scope" ‚Äî this stage makes the
code do what the comment says.

#### B.3.1 The pure egress module ‚Äî `crates/runtime-main/src/import/egress.rs` (new)

```rust
//! Import-fetch egress security ‚Äî SSRF defense for the user-supplied
//! import URL (M07.5 / ADR-0018).
//!
//! `classify_ip` + `check_url` are PURE ‚Äî fully unit-tested. DNS
//! resolution is behind the injected `Resolver` seam (the `Fetcher` /
//! `Sandbox` / `Clock` archetype) so tests inject "this host resolves
//! to 127.0.0.1" and assert rejection without touching the network.
//! Only the real syscalls (the `reqwest` GET, the DNS lookup) live in
//! `import/fetch.rs`, the runtime-main OS-call coverage holdout.

use std::net::{IpAddr, SocketAddr};

use reqwest::Url;

use super::{Fetcher, ImportError};

/// Largest import artifact body accepted (defends against an
/// over-large / decompression-bomb response).
pub const MAX_BODY_BYTES: usize = 8 * 1024 * 1024;

/// Largest redirect chain followed before the fetch is abandoned.
pub const MAX_REDIRECTS: usize = 5;

/// Why an egress request was refused.
#[derive(Debug, Clone, PartialEq, Eq)]
pub enum EgressReject {
    /// The URL did not parse.
    Parse(String),
    /// The scheme is not `https`.
    Scheme(String),
    /// The URL has no host component.
    NoHost,
    /// A resolved address is not publicly routable (SSRF defense).
    PrivateAddress(IpAddr),
    /// The host resolved to no addresses.
    NoAddress(String),
}

/// PURE ‚Äî classify a resolved IP. `Ok` only for a publicly-routable
/// address; `Err` for every internal / special range.
///
/// IPv4-mapped IPv6 (`::ffff:a.b.c.d`) is unwrapped to its v4 form
/// FIRST ‚Äî a mapped private address is a classic SSRF bypass. The
/// CGNAT (`100.64/10`), IPv6 ULA (`fc00::/7`), and IPv6 link-local
/// (`fe80::/10`) ranges are hand-coded because `std`'s predicates for
/// them are unstable on the project MSRV (ADR-0018 ¬ßNotes).
///
/// # Errors
///
/// [`EgressReject::PrivateAddress`] for any non-public address.
pub fn classify_ip(ip: IpAddr) -> Result<(), EgressReject> {
    // Unwrap IPv4-mapped IPv6 so a mapped private address classifies
    // as the v4 address it really is.
    let normalized = match ip {
        IpAddr::V4(v4) => IpAddr::V4(v4),
        IpAddr::V6(v6) => match v6.to_ipv4_mapped() {
            Some(v4) => IpAddr::V4(v4),
            None => IpAddr::V6(v6),
        },
    };
    let blocked = match normalized {
        IpAddr::V4(v4) => {
            v4.is_loopback()
                || v4.is_private()
                || v4.is_link_local()
                || v4.is_unspecified()
                || v4.is_broadcast()
                || v4.is_multicast()
                || v4.is_documentation()
                // CGNAT shared address space 100.64.0.0/10.
                || (v4.octets()[0] == 100 && (v4.octets()[1] & 0xc0) == 64)
        }
        IpAddr::V6(v6) => {
            v6.is_loopback()
                || v6.is_unspecified()
                || v6.is_multicast()
                // Unique local fc00::/7.
                || (v6.segments()[0] & 0xfe00) == 0xfc00
                // Link-local unicast fe80::/10.
                || (v6.segments()[0] & 0xffc0) == 0xfe80
        }
    };
    if blocked {
        Err(EgressReject::PrivateAddress(normalized))
    } else {
        Ok(())
    }
}

/// PURE ‚Äî parse the raw URL and enforce the `https`-only scheme
/// allowlist. `reqwest::Url` is the WHATWG parser ‚Äî it canonicalizes
/// IPv4 encoding tricks (`0x7f000001`, `2130706433`) to a normal host,
/// so a later [`classify_ip`] sees the real address.
///
/// # Errors
///
/// [`EgressReject::Parse`] / [`EgressReject::Scheme`] /
/// [`EgressReject::NoHost`].
pub fn check_url(raw: &str) -> Result<Url, EgressReject> {
    let url = Url::parse(raw).map_err(|e| EgressReject::Parse(e.to_string()))?;
    if url.scheme() != "https" {
        return Err(EgressReject::Scheme(url.scheme().to_string()));
    }
    if url.host_str().is_none() {
        return Err(EgressReject::NoHost);
    }
    Ok(url)
}
```

```rust
/// DNS-resolution seam ‚Äî the real impl is [`super::fetch::SystemResolver`]
/// (`tokio::net::lookup_host`); tests inject a fake mapping host ‚Üí
/// addresses. Async because the real path is a syscall.
#[async_trait::async_trait]
pub trait Resolver: Send + Sync {

    /// addresses.
    ///
    /// # Errors
    ///
    /// The resolver failure, stringified.
    async fn resolve(&self, host: &str) -> Result<Vec<IpAddr>, String>;

/// A URL that passed egress validation, with the address the HTTP
/// connection is pinned to (DNS-rebinding defense ‚Äî the client must
/// not re-resolve).
#[derive(Debug, Clone)]
pub struct ValidatedTarget {
    /// The parsed, scheme-checked URL.
    pub url: Url,
    /// The host string (for the `Host:` header / DNS-pin key).
    pub host: String,
    /// The validated address the connection pins to.
    pub addr: SocketAddr,
}

#[derive(Debug)]
```

`validate_egress` composes the pure checks with the `Resolver` seam ‚Äî
it is the single egress decision point `fetch_with` calls for the
initial URL **and** for every redirect target:

```rust
/// Validate a URL for egress: parse + `https` + resolve + classify
/// EVERY resolved address + pin. A single non-public address fails the
/// whole request.
///
/// # Errors
///
/// [`ImportError::Fetch`] carrying the [`EgressReject`] reason.
pub async fn validate_egress(
    raw: &str,
    resolver: &dyn Resolver,
) -> Result<ValidatedTarget, ImportError> {
    let url = check_url(raw).map_err(reject_to_import)?;
    // check_url guarantees a host; strip IPv6 brackets for the resolver.
    let host = url
        .host_str()
        .expect("check_url ensured a host")
        .trim_start_matches('[')
        .trim_end_matches(']')
        .to_string();
    let port = url.port_or_known_default().unwrap_or(443);

    let ips = resolver
        .resolve(&host)
        .await
        .map_err(|e| ImportError::Fetch(format!("DNS resolution failed: {e}")))?;
    if ips.is_empty() {
        return Err(reject_to_import(EgressReject::NoAddress(host)));
    }
    // Reject if ANY resolved address is non-public (a host that
    // resolves to both public and private is treated as hostile).
    for ip in &ips {
        classify_ip(*ip).map_err(reject_to_import)?;
    }
    Ok(ValidatedTarget {
        url,
        host,
        addr: SocketAddr::new(ips[0], port),
    })
}

/// Map an [`EgressReject`] onto the pipeline's [`ImportError::Fetch`]
/// (consistent with how the old `NetworkGate` denial surfaced ‚Äî no new
/// `ImportError` variant).
fn reject_to_import(r: EgressReject) -> ImportError {
    ImportError::Fetch(format!("egress blocked: {r:?}"))
}
```

```rust
#[async_trait::async_trait]
impl Fetcher for HttpFetcher {
    async fn fetch_hop(&self, target: &ValidatedTarget) -> Result<FetchHop, String> {
        let client = reqwest::Client::builder()
            // No auto-follow ‚Äî fetch_with re-validates each hop.
            .redirect(reqwest::redirect::Policy::none())
            .connect_timeout(std::time::Duration::from_secs(10))
            .timeout(std::time::Duration::from_secs(30))
            // DNS-pin: connect to the validated address, not a fresh
            // resolution (DNS-rebinding defense).
            .resolve(&target.host, target.addr)
            .build()
            .map_err(|e| e.to_string())?;

        let resp = client
            .get(target.url.clone())
            .send()
            .await
            .map_err(|e| e.to_string())?;
        let status = resp.status();

        if status.is_redirection() {
            let loc = resp
                .headers()
                .get(reqwest::header::LOCATION)
                .and_then(|v| v.to_str().ok())
                .ok_or_else(|| format!("{status}: redirect with no Location"))?;
            // Resolve a relative Location against the current URL.
            let next = target.url.join(loc).map_err(|e| e.to_string())?;
            return Ok(FetchHop::Redirect(next.to_string()));
        }
        if !status.is_success() {
            return Err(format!("HTTP {status}"));
        }

        // Streamed, capped body read ‚Äî robust against a lying
        // Content-Length.
        let mut resp = resp;
        let mut body = Vec::new();
        while let Some(chunk) = resp.chunk().await.map_err(|e| e.to_string())? {
            body.extend_from_slice(&chunk);
            if body.len() > MAX_BODY_BYTES {
                return Err(format!("response body exceeds {MAX_BODY_BYTES} bytes"));
            }
        }
        Ok(FetchHop::Body(body))
    }
}
```

```rust
/// Fetch the artifact bytes.
///
/// File reads are local. URL fetches are SSRF-hardened (ADR-0018):
/// each hop ‚Äî the initial URL and every redirect target ‚Äî is
/// re-validated through `egress::validate_egress` (https-only,
/// public-address-only, DNS-pinned) before the GET.
///
/// # Errors
///
/// [`ImportError::Fetch`] for a missing file, an egress rejection,
/// a transport failure, or too many redirects.
pub async fn fetch_with(
    src: &ImportSource,
    get: &dyn Fetcher,
    resolver: &dyn egress::Resolver,
) -> Result<Vec<u8>, ImportError> {
    match src {
        ImportSource::File(p) => {
            std::fs::read(p).map_err(|e| ImportError::Fetch(e.to_string()))
        }
        ImportSource::Url(u) => {
            let mut current = u.clone();
            for _ in 0..=egress::MAX_REDIRECTS {
                let target = egress::validate_egress(&current, resolver).await?;
                match get
                    .fetch_hop(&target)
                    .await
                    .map_err(ImportError::Fetch)?
                {
                    egress::FetchHop::Body(bytes) => return Ok(bytes),
                    egress::FetchHop::Redirect(next) => current = next,
                }
            }
            Err(ImportError::Fetch(format!(
                "too many redirects (max {})",
                egress::MAX_REDIRECTS
            )))
        }
    }
}
```

```ts
 * Runs the install half the backend held back; resolves to the
 * terminal `installed` outcome.

export async function completeImportArtifact(
  pendingReviewId: string,
): Promise<ImportOutcome> {
  return await invoke<ImportOutcome>('complete_import_artifact', { pendingReviewId });
```
