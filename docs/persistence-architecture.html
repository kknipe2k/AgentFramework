How to read it

  • Layer 1 — Static Contract. The spec, schemas, and reference frameworks. Changes only via an ADR.
  • Layer 2 — Slow-Evolving Protocol. CLAUDE.md, MVP doc, templates. Auto-loaded every session; updated when the protocol itself shifts.
  • Layer 3 — Per-Milestone Prompt. One document per parent milestone (M01, M02, …, M11). Stages A–E live inside.
  • Layer 4 — Per-Session Memory. Ephemeral. The Claude Code context window for one stage. Cleared between stages.
  • Layer 5 — Stage Outputs. What survives the stage and feeds the next: code commits, retrospectives, gap-analysis entries (append-only), changelog.

Every layer below reads from the layers above. The session is the only mutable layer; outputs (Layer 5) get committed and become the next session's read material.
